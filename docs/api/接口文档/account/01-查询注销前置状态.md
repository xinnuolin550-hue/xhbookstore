# 查询注销前置状态

> **接口路径**: `GET /api/mp/v1/account/cancel-eligibility`
> **认证要求**: 是（需在 Header 中携带 `Authorization: Bearer <accessToken>`）
> **Content-Type**: 无（GET 请求）
> **对应代码**: `AccountController.java`

## 接口说明

在发起账号注销前，查询当前账号是否满足注销条件。系统会检查是否存在未归还的图书、是否为有效员工身份等限制条件，避免用户盲目发起注销。

---

## 请求参数

无请求体。需在请求头中携带认证信息。

### 请求头

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| Authorization | String | 是 | 无 | Bearer 认证，格式：`Bearer <accessToken>` |

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 提示信息 |
| data.canCancel | Boolean | 是否允许注销（true=可以/false=被拦截） |
| data.blockedReason | String | 拦截原因（canCancel=true 时为 null） |
| data.hasUnreturnedBooks | Boolean | 是否存在未归还的图书 |
| data.isStaffActive | Boolean | 员工身份是否仍有效 |
| data.currentPoints | Integer | 当前剩余积分 |
| requestId | String | 本次请求唯一标识 |

## 业务逻辑

1. 根据当前登录用户查询借阅记录，判断是否存在 `status != RETURNED` 的借书单
2. 查询员工身份状态，判断是否关联有效员工记录
3. `blockedReason` 在 `canCancel=false` 时返回具体拦截原因的文本描述（多原因时拼接）
4. 当前积分用于提示用户注销前可考虑消耗积分，不影响注销判断
5. 该接口为查询接口，不产生任何数据变更
