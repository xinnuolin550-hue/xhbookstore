# 注销账号

> **接口路径**: `POST /api/mp/v1/account/cancel`
> **认证要求**: 是（需在 Header 中携带 `Authorization: Bearer <accessToken>`）
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `AccountController.java`

## 接口说明

发起账号注销申请。注销前必须先调用「查询注销前置状态」接口确认允许注销，否则请求会被拒绝。注销操作不可逆，注销后账号数据将被标记删除或物理清除。

---

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| reason | String | 否 | 无 | 注销原因（如："不再使用"、"重复注册"等） |
| confirmRead | Boolean | 是 | 无 | 确认已阅读注销须知（必须为 true） |

## 请求报文

```json
{
  "reason": "不再使用本店服务",
  "confirmRead": true
}
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 提示信息 |
| data.success | Boolean | 注销结果（true=已受理） |
| data.cancelledAt | Long | 注销操作时间戳（毫秒） |
| requestId | String | 本次请求唯一标识 |

## 成功响应

## 错误响应

### 不符合注销条件（需先调用 cancel-eligibility 查询）

```json
{
  "code": 60001,
  "message": "当前账号不允许注销",
  "data": null,
  "requestId": "req_20260630120502"
}
```

### 存在未归还图书

```json
{
  "code": 60002,
  "message": "存在未还图书，无法注销",
  "data": null,
  "requestId": "req_20260630120503"
}
```

### 员工身份仍有效

```json
{
  "code": 60003,
  "message": "员工身份仍有效，无法注销",
  "data": null,
  "requestId": "req_20260630120504"
}
```

### 未确认注销须知

```json
{
  "code": 400,
  "message": "请先确认已阅读注销须知",
  "data": null,
  "requestId": "req_20260630120505"
}
```

### 未提供 Token 或 Token 无效

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630120506"
}
```

## 业务逻辑

1. 校验 `confirmRead` 必须为 `true`，否则拒绝请求
2. 内部重新校验注销条件（与 cancel-eligibility 逻辑一致），不满足时返回对应错误码
3. 满足条件后执行注销：**标记会员账号为已注销状态**，关联数据视业务规则处理
4. 注销操作不可逆，需在前端引导用户确认风险
5. 注销后该手机号可重新注册为新账号
