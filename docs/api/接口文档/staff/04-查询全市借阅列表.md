# 查询全市借阅列表

> **接口路径**: `GET /api/mp/v1/staff/borrows`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `borrowsList()`

## 接口说明

员工查询当前门店（全市范围）的借阅订单列表，可按手机号和借阅状态过滤，支持分页。用于员工管理界面的借阅记录总览。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| phone | String | 否 | — | 会员手机号（精确匹配或模糊搜索） |
| status | String | 否 | — | 借阅状态：`0` 借阅中 / `1` 已归还 / `2` 部分归还 |
| pageNo | int | 否 | 1 | 当前页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页条数（最大 100） |

## 请求报文

无请求体。请求示例：

```
GET /api/mp/v1/staff/borrows?phone=138****1234&status=0&pageNo=1&pageSize=20
```

## curl 测试

### 示例 1：无过滤条件（查询全部）

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/borrows?pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

### 示例 2：按手机号 + 状态过滤

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/borrows?phone=138&status=0&pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

## 业务逻辑

1. `phone` 参数支持模糊搜索，根据手机号前缀或完整号码匹配关联的会员借阅记录。
2. `status` 过滤条件：
   - `0` — 查询借阅中（未全部归还）的订单
   - `1` — 查询已全部归还的订单
   - `2` — 查询部分归还的订单
   - 不传 — 查询所有状态的订单
3. 该接口查询范围为**当前门店（全市）**的借阅数据，需跨会员查询。
4. 返回列表按 `borrowTime` 降序排列，最新的借阅在前。
5. `details` 数组中每项提供图书维度的借阅状态，用于员工快速查看每本书的归还进度。
6. 分页参数 `pageNo` 和 `pageSize` 均有默认值，前端可省略。
