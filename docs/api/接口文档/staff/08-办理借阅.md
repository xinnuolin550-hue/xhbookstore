# 办理借阅

> **接口路径**: `POST /api/mp/v1/staff/members/{memberId}/borrows`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `borrow()`

## 接口说明

为指定会员办理图书借阅。员工选择多本图书后提交，系统创建借阅订单并记录借阅明细。借阅操作涉及库存校验和会员借阅资格校验。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| memberId | String | 是（路径参数） | — | 会员 ID，如 `1` |
| books | Array | 是 | — | 借阅图书列表，至少需要一本图书 |
| books[].bookId | Integer | 是 | — | 图书 ID |
| books[].bookTitle | String | 是 | — | 图书名称 |
| books[].bookIsbn | String | 是 | — | 图书 ISBN 编号 |
| remark | String | 否 | null | 借阅备注 |

## 请求报文

```json
{
  "books": [
    {
      "bookId": 2001,
      "bookTitle": "JavaScript高级程序设计",
      "bookIsbn": "9787115275790"
    },
    {
      "bookId": 2002,
      "bookTitle": "深入浅出Node.js",
      "bookIsbn": "9787115500007"
    }
  ],
  "remark": "员工借阅"
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/members/1/borrows \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{"books":[{"bookId":2001,"bookTitle":"JavaScript高级程序设计","bookIsbn":"9787115275790"},{"bookId":2002,"bookTitle":"深入浅出Node.js","bookIsbn":"9787115500007"}],"remark":"员工借阅"}'
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 借阅订单数据 |
| data.borrowOrderNo | String | 借书单号，格式如 `DY20260630143000123456` |
| data.totalBookCount | int | 本次借阅总数量 |
| data.borrowTime | Long | 借阅时间戳（毫秒） |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. 员工需先通过扫码或搜索获取会员信息，传入 `memberId`。
2. `books` 数组至少需要一本图书，否则返回 `BORROW_BOOK_REQUIRED`（40002）。
3. 每本书需提供 `bookId`、`bookTitle` 和 `bookIsbn`，用于创建借阅明细。
4. 后端会校验会员是否存在、会员卡状态是否有效以及借阅资格；若不满足则返回 `BORROW_DENIED`（40001）。
5. 借书单号格式为 `DY` + 日期时间 + 序列号，全局唯一。
6. `remark` 字段可选，用于记录员工备注或借阅原因。
7. 操作员工信息从 `request.getAttribute("userId")` 获取，若为空则默认 `system`。
