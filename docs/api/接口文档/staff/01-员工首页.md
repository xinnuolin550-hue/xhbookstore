# 员工首页

> **接口路径**: `GET /api/mp/v1/staff/home`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `home()`

## 接口说明

展示员工端首页核心数据，包括门店名称、今日门店借阅量、今日员工个人借阅量。

## 请求参数

无请求参数。

## 请求报文

无请求体。

## curl 测试

```bash
curl -X GET http://localhost:8091/api/mp/v1/staff/home \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 首页数据 |
| data.storeName | String | 门店名称 |
| data.todayStoreBorrowCount | int | 今日全店借阅笔数 |
| data.todayStaffBorrowCount | int | 今日员工个人借阅笔数 |
| requestId | String | 请求唯一标识 |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "storeName": "新华书店总店",
    "todayStoreBorrowCount": 12,
    "todayStaffBorrowCount": 3
  },
  "requestId": "req_001a2b3c"
}
```

## 错误响应

### 未认证（Token 缺失或无效）

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_001a2b3c"
}
```

### 无权限访问

```json
{
  "code": 403,
  "message": "无权限访问",
  "data": null,
  "requestId": "req_001a2b3c"
}
```

## 业务逻辑

1. 员工调用前需通过微信小程序登录获取 `accessToken`，并在请求头中携带 `Authorization: Bearer <accessToken>`。
2. `todayStoreBorrowCount` 统计当前门店当日所有员工的借阅数（含本人）。
3. `todayStaffBorrowCount` 仅统计当前登录员工当日的借阅数。
4. 门店名称由当前员工所属门店动态决定。
5. 返回数据不涉及数据库查询，为轻量快速接口。
