# 查询指定会员借阅记录

> **接口路径**: `GET /api/mp/v1/staff/members/{memberId}/borrows`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `memberBorrows()`

## 接口说明

根据会员 ID 查询该会员的全部或当前在借的借阅记录。支持按模式（全部/当前在借）、借阅状态过滤，分页返回。常用于员工查看会员借阅历史和管理借还书操作。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| memberId | String | 是（路径参数） | — | 会员唯一 ID |
| mode | String | 否 | — | 查询模式：`all` 全部记录 / `current` 仅当前在借 |
| status | String | 否 | — | 借阅状态：`0` 借阅中 / `1` 已归还 / `2` 部分归还 |
| pageNo | int | 否 | 1 | 当前页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页条数（最大 100） |

### 参数说明

- `mode=current` 时，仅返回 `borrowStatus` 为 `0`（借阅中）或 `2`（部分归还）且未全部完成的订单。
- `mode=all` 或不传 `mode` 时，返回该会员所有借阅记录。
- `status` 可与 `mode` 结合使用，进一步过滤特定状态的记录。

## 请求报文

无请求体。请求示例：

```
GET /api/mp/v1/staff/members/1/borrows?mode=all&pageNo=1&pageSize=20
```

## curl 测试

### 示例 1：查询全部记录

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/members/1/borrows?mode=all&pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

### 示例 2：仅查询当前在借记录

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/members/1/borrows?mode=current&pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

### 示例 3：按状态过滤已归还记录

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/members/1/borrows?mode=all&status=1&pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

## 业务逻辑

1. 根据 `memberId` 查询会员是否存在，不存在则返回 `MEMBER_NOT_FOUND`。
2. 查询该会员的所有借阅订单，按 `borrowTime` 降序排列。
3. `mode` 参数过滤逻辑：
   - `mode=current` — 仅筛选 `borrowStatus` 为 `0`（借阅中）或 `2`（部分归还）的订单。
   - `mode=all` 或不传 — 返回所有状态的订单。
4. `status` 参数进一步过滤：如 `status=1` 仅查看已归还的订单。
5. 每个订单附带 `details` 明细数组，展示每本图书的借阅详情，供员工逐本确认。
6. `member` 概要信息包括脱敏手机号、当前积分等，方便员工在操作时快速了解会员状态。
7. 分页由后端处理，返回 `page` 对象包含 `hasMore` 字段，前端据此判断是否展示"加载更多"。
