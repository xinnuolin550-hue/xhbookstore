# 消耗积分

> **接口路径**: `POST /api/mp/v1/staff/members/{memberId}/points/deduct`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `deductPoints()`

## 接口说明

消耗指定会员的积分（扣减操作）。操作前会校验会员积分余额是否充足，若不足则返回错误。消耗成功后返回操作前后的积分快照，积分变动值为负数。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| memberId | String | 是（路径参数） | — | 会员 ID，如 `1` |
| reasonId | String | 是 | — | 积分事项 ID，如 `"3"`（积分兑换） |
| points | Integer | 是 | — | 消耗积分值，必须为正整数（表示要扣除的数值） |
| remark | String | 否 | null | 操作备注 |

## 请求报文

```json
{
  "reasonId": "3",
  "points": 100,
  "remark": "积分兑换礼品"
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/members/1/points/deduct \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{"reasonId":"3","points":100,"remark":"积分兑换礼品"}'
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 操作结果数据 |
| data.success | boolean | 操作是否成功 |
| data.pointsRecordId | String | 积分记录编号，格式如 `OT20260630150000123456`（OT 前缀表示消耗） |
| data.beforePoints | int | 操作前积分余额 |
| data.pointsDelta | int | 积分变动值（负数，表示消耗） |
| data.afterPoints | int | 操作后积分余额 |
| data.operatedAt | Long | 操作时间戳（毫秒） |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. **必要参数校验**：`reasonId` 不能为空（否则返回 `POINTS_REASON_INVALID` 30003），`points` 不能为空且必须是正整数（否则返回参数错误 400）。注意前端传递的是要消耗的积分值（正数），后端会转成负数记录。
2. **余额校验**：操作前会查询会员当前积分余额，若 `currentPoints < points` 则返回 `POINTS_NOT_ENOUGH`（30001）。
3. **会员存在性校验**：先查询会员信息，若会员不存在则返回 `MEMBER_NOT_FOUND`（20001）。
4. **积分记录前缀**：消耗积分记录以 `OT` 前缀开头，后跟时间戳，与增加积分的 `IN` 前缀区分。
5. **pointsDelta 为负数**：响应中的 `pointsDelta` 为负数（如 `-100`），表示积分被消耗，前端可直接展示。
6. **当前实现状态**：接口已实现基本的参数校验和余额校验逻辑，完整的扣减持久化（插入 OUT 订单、插入出账单等）标记为 `TODO`，当前直接返回成功响应并模拟操作后数据。
7. 操作员工信息从 `request.getAttribute("userId")` 获取，若为空则默认 `system`。
