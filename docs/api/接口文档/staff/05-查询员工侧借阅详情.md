# 查询员工侧借阅详情

> **接口路径**: `GET /api/mp/v1/staff/borrows/{borrowId}`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `borrowDetail()`

## 接口说明

根据借书单号查询完整的借阅详情，包含订单主信息、借阅明细列表和归还记录列表。用于员工查看某笔借阅的完整操作轨迹。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| borrowId | String | 是（路径参数） | — | 借书单号，格式如 `DY20260630143000123456` |

## 请求报文

无请求体。请求示例：

```
GET /api/mp/v1/staff/borrows/DY20260630143000123456
```

## curl 测试

```bash
curl -X GET http://localhost:8091/api/mp/v1/staff/borrows/DY20260630143000123456 \
  -H "Authorization: Bearer <accessToken>"
```

## 业务逻辑

1. 根据 `borrowId`（即 `orderNo`）查询借阅订单主表，若找不到匹配记录则返回 `NOT_FOUND`。
2. 查询该订单下的所有借阅明细（`details`），展示每本图书的借阅进度。
3. 查询该订单下的所有归还记录（`returns`），按 `returnTime` 升序排列。
4. `returns` 数组中的 `returnType` 含义：
   - `0` — 正常归还
   - `1` — 损坏归还
   - `2` — 丢失（需赔偿）
5. 返回数据中的时间字段由 Jackson 的 `@JsonFormat` 注解格式化为 `yyyy-MM-dd HH:mm:ss` 字符串。
6. 当订单全部归还后，`order.returnAllTime` 不为 null，`order.borrowStatus` 变为 `1`（已归还），`order.isFinished` 变为 `1`。
