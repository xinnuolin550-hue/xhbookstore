# 增加积分

> **接口路径**: `POST /api/mp/v1/staff/members/{memberId}/points/add`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `addPoints()`

## 接口说明

为指定会员增加积分。该接口采用**悲观锁 + 事务**机制保证积分操作的原子性和数据一致性，适用于并发场景下的积分增加操作。操作成功后返回操作前后的积分快照。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| memberId | String | 是（路径参数） | — | 会员 ID，如 `1` |
| reasonId | String | 是 | — | 积分事项 ID，如 `"1"`（活动赠送） |
| points | Integer | 是 | — | 增加积分值，必须为正整数 |
| remark | String | 否 | "员工操作" | 操作备注 |

## 请求报文

```json
{
  "reasonId": "1",
  "points": 50,
  "remark": "活动赠送积分"
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/members/1/points/add \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{"reasonId":"1","points":50,"remark":"活动赠送积分"}'
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 操作结果数据 |
| data.success | boolean | 操作是否成功 |
| data.pointsRecordId | String | 积分记录编号，格式如 `IN20260628120000123456`（IN 前缀表示增加） |
| data.beforePoints | int | 操作前积分余额 |
| data.pointsDelta | int | 积分变动值（正数） |
| data.afterPoints | int | 操作后积分余额 |
| data.operatedAt | Long | 操作时间戳（毫秒） |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. **必要参数校验**：`reasonId` 不能为空（否则返回 `POINTS_REASON_INVALID` 30003），`points` 不能为空且必须是正整数（否则返回参数错误 400）。
2. **悲观锁+事务**：调用 `pointsService.addPoints()` 时，内部使用 `SELECT ... FOR UPDATE` 悲观锁锁定会员积分记录，防止并发情况下积分数据不一致。整个操作在一个数据库事务内完成。
3. **积分事项校验**：`reasonId` 必须是有效的增加积分事项（如"活动赠送"、"借阅奖励"），否则返回 `POINTS_REASON_INVALID`。
4. **单次最大限制**：`points` 值不能超过 `maxPoints`（当前为 99999），否则返回 `POINTS_EXCEED_MAX`（30002）。
5. **积分记录编号**：增加积分记录以 `IN` 前缀开头，后跟日期时间和序列号，全局唯一。
6. **操作前后快照**：响应中返回 `beforePoints` 和 `afterPoints`，便于前端展示积分增减前后的变化。
7. `remark` 为空时默认赋值为 `"员工操作"`。
8. 操作员工信息从 `request.getAttribute("userId")` 获取，若为空则默认 `system`。
