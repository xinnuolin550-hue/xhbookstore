# 查询扫码会员概要

> **接口路径**: `GET /api/mp/v1/staff/members/{memberId}/overview`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `memberOverview()`

## 接口说明

扫码解析会员码获取 `memberId` 后，调用此接口查询会员概要信息，展示在员工操作界面上。包含会员基本信息、会员卡信息以及当前可操作权限（借书、还书、积分增减等）。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| memberId | String | 是（路径参数） | — | 会员唯一 ID |

## 请求报文

无请求体。请求示例：

```
GET /api/mp/v1/staff/members/1/overview
```

## curl 测试

```bash
curl -X GET http://localhost:8091/api/mp/v1/staff/members/1/overview \
  -H "Authorization: Bearer <accessToken>"
```

## 业务逻辑

1. 根据 `memberId` 查询会员主表数据。
2. 若会员不存在，抛出 `MEMBER_NOT_FOUND` 异常，前端应引导员工重新扫码。
3. `phoneDisplay` 对手机号中间四位脱敏，仅显示前三位和后四位（如 `138****1234`）。
4. `memberLevel` 由 `levelId * 10` 计算得出，范围 0-100。
5. `cardStatus` 根据会员卡有效期和状态字段动态计算：
   - 当前时间 < `effectiveAt` → `"inactive"`
   - 当前时间 > `expiredAt` → `"expired"`
   - `status` 字段置为禁用 → `"disabled"`
   - 正常 → `"active"`
6. `availability` 中的各项 `can*` 字段根据会员卡状态、当前借阅数量上限、积分规则等综合判断，返回 `true`/`false` 及对应的禁用原因。
7. `maxAddPoints` 和 `maxDeductPoints` 受系统配置的积分单次操作上限和会员等级共同约束。
