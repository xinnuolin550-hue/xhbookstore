# 办理还书

> **接口路径**: `POST /api/mp/v1/staff/borrow-returns`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `returnBooks()`

## 接口说明

员工为会员办理还书操作。支持批量归还，每一本归还的图书可标记归还状态（正常 / 损坏 / 丢失）并填写备注。系统会自动生成归还单号并记录操作员工信息。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| borrowOrderNo | String | 是 | — | 借书单号，格式如 `DY20260630143000123456` |
| returnItems | Array | 是 | — | 归还图书列表，至少一项 |
| returnItems[].borrowDetailId | Long | 是 | — | 借阅明细 ID（对应借阅详情中的 `id` 字段） |
| returnItems[].bookId | Long | 否 | — | 图书 ID |
| returnItems[].returnStatus | String | 是 | — | 归还状态：`normal` 正常 / `damaged` 损坏 / `lost` 丢失 |
| returnItems[].remark | String | 否 | — | 备注说明 |

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 返回数据 |
| data.returnOrderNo | String | 归还单号 |
| data.returnedCount | int | 本次成功归还数量 |
| data.totalCount | int | 借书单总数量（用于进度展示） |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. **参数校验**：校验 `borrowOrderNo` 不为空，`returnItems` 不为空且至少一项。
2. **鉴权**：从请求属性中获取当前操作员工 ID（`userId`），若为空则记录为 `system`。
3. **业务校验**：
   - 检查借书单是否存在且状态可归还。
   - 校验每本图书是否属于该借书单且未全部归还。
   - 若某本图书已全部归还，返回 `BORROW_ALREADY_RETURNED`（40005）。
   - 若借书单状态不允许还书（如已全部归还），返回 `BORROW_RETURN_DENIED`（40004）。
4. **归还处理**：
   - `returnStatus` 为 `normal` 时，更新明细的 `returnedQty`，标记为正常归还。
   - `returnStatus` 为 `damaged` 时，标记为损坏归还，记录备注。
   - `returnStatus` 为 `lost` 时，标记为丢失，可能需要触发赔偿流程。
5. **归还单生成**：自动生成归还单号（格式 `RT` + 时间戳 + 随机码），记录所有归还明细。
6. **订单状态更新**：
   - 若本次归还后仍有未还图书，订单 `borrowStatus` 更新为 `2`（部分归还）。
   - 若全部图书均已归还，订单 `borrowStatus` 更新为 `1`（已归还），`isFinished` 更新为 `1`。
7. **返回数据**：`returnedCount` 表示本次操作成功归还的图书数量，`totalCount` 表示借书单的总图书数量。
