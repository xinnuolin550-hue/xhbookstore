# 查询全市积分列表

> **接口路径**: `GET /api/mp/v1/staff/points-records`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `pointsRecordsList()`

## 接口说明

查询全市（跨会员）积分变动记录列表，支持按手机号、会员 ID 和积分方向筛选。当传入 `memberId` 时查询指定会员的积分记录；不传时理论上查询全市记录（当前仅实现按会员查询，全市查询为 TODO）。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| phone | String | 否 | — | 会员手机号（模糊搜索，当前暂未支持） |
| memberId | String | 否 | — | 会员 ID，传入时查询指定会员的记录 |
| direction | String | 否 | — | 积分方向：`add`（增加）或 `deduct`（消耗） |
| pageNo | int | 否 | 1 | 页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页条数 |

## 请求报文

无请求体。查询示例：

```
GET /api/mp/v1/staff/points-records?memberId=1&direction=add&pageNo=1&pageSize=20
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 分页数据 |
| data.page | object | 分页对象 |
| data.page.list | Array | 积分记录列表 |
| data.page.list[].pointsRecordId | String | 积分记录编号，`IN` 前缀为增加，`OT` 前缀为消耗 |
| data.page.list[].reasonName | String | 积分事项名称（如"活动赠送"、"积分兑换"） |
| data.page.list[].direction | String | 积分方向：`add`（增加）或 `deduct`（消耗） |
| data.page.list[].pointsDelta | int | 积分变动值（正数表示增加，负数表示消耗） |
| data.page.list[].beforePoints | int | 操作前积分余额 |
| data.page.list[].afterPoints | int | 操作后积分余额 |
| data.page.list[].operatedAt | Long | 操作时间戳（毫秒） |
| data.page.list[].staffName | String | 操作员工名称，小程序端记录为"小程序" |
| data.page.pageNo | int | 当前页码 |
| data.page.pageSize | int | 每页条数 |
| data.page.totalRow | int | 总记录数 |
| data.page.totalPage | int | 总页数 |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. **memberId 筛选**：当 `memberId` 参数不为空时，查询该会员的所有积分记录（包含增加和消耗），按创建时间降序排列。
2. **全市查询 TODO**：当不传 `memberId` 时，理论上应查询全部门店的积分变动记录，当前仅返回空列表（`Collections.emptyList()`）。
3. **direction 筛选**：`direction` 参数当前未在服务端做过滤（参数已声明但未使用）；积分方向的区分可通过 `pointsRecordId` 前缀实现：`IN` 开头为增加（add），`OT` 开头为消耗（deduct）。
4. **staffName 说明**：员工端操作的记录 `staffName` 为操作员工名称；通过小程序端产生的记录显示为"小程序"。
5. **分页**：默认每页 20 条，页码从 1 开始。
6. `pointsDelta` 正负含义：增加记录为正数，消耗记录为负数。
