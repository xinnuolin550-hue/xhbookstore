# 查询积分详情

> **接口路径**: `GET /api/mp/v1/staff/points-records/{pointsRecordId}`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `pointsDetail()`

## 接口说明

根据积分记录编号查询单条积分变动的详细信息。当前实现为最小可用版本，仅返回 `pointsRecordId`，完整的积分详情（含变动前后余额、操作员工、时间等）功能为 TODO，待后续完善。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pointsRecordId | String | 是（路径参数） | — | 积分记录编号，如 `IN20260628120000123456` |

## 请求报文

无请求体。请求示例：

```
GET /api/mp/v1/staff/points-records/IN20260628120000123456
```

## curl 测试

### 查询增加积分详情

```bash
curl -X GET http://localhost:8091/api/mp/v1/staff/points-records/IN20260628120000123456 \
  -H "Authorization: Bearer <accessToken>"
```

### 查询消耗积分详情

```bash
curl -X GET http://localhost:8091/api/mp/v1/staff/points-records/OT20260630150000123456 \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 积分详情数据 |
| data.pointsRecordId | String | 积分记录编号 |
| requestId | String | 请求唯一标识 |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "pointsRecordId": "IN20260628120000123456"
  },
  "requestId": "req_00df5g6h"
}
```

## 错误响应

### 记录不存在（404）

```json
{
  "code": 404,
  "message": "资源不存在",
  "data": null,
  "requestId": "req_00df5g6h"
}
```

### 未认证（Token 缺失或无效） — code 401

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_00df5g6h"
}
```

## 业务逻辑

1. **当前实现为最小可用版本**：该接口仅返回传入的 `pointsRecordId`，未查询数据库，未返回完整的积分变动详情。
2. **TODO 待完善**：后续应补充以下完整信息：
   - 会员信息（会员 ID、姓名、会员编号）
   - 积分变动类型（增加/消耗）和事项名称
   - 变动前后积分余额（`beforePoints` / `afterPoints`）
   - 积分变动值（`pointsDelta`）
   - 操作时间（`operatedAt`）
   - 操作员工信息（`staffName`、`staffId`）
   - 备注信息
3. **记录编号前缀**：`IN` 开头为增加积分记录，`OT` 开头为消耗积分记录，可用于前端区分展示。
4. **异常情况**：若传入不存在的 `pointsRecordId`，待完整实现后应返回 `NOT_FOUND`（404）。
