# 查询积分事项

> **接口路径**: `GET /api/mp/v1/staff/points-reasons`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `pointsReasons()`

## 接口说明

查询可用的积分变动事项（原因）列表。根据 `direction` 参数分别返回增加积分和消耗积分时可选的事项列表，供前端下拉选择。每个事项包含默认积分值和启用状态。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| direction | String | 是 | — | 积分方向：`add`（增加）或 `deduct`（消耗） |
| memberId | String | 否 | — | 会员 ID，用于后续根据会员等级或标签过滤可用事项（当前暂未实现） |

## 请求报文

无请求体。查询示例：

```
GET /api/mp/v1/staff/points-reasons?direction=add&memberId=1
```

## curl 测试

### 查询增加积分事项

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/points-reasons?direction=add&memberId=1" \
  -H "Authorization: Bearer <accessToken>"
```

### 查询消耗积分事项

```bash
curl -X GET "http://localhost:8091/api/mp/v1/staff/points-reasons?direction=deduct&memberId=1" \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 积分事项数据 |
| data.list | Array | 积分事项列表 |
| data.list[].reasonId | String | 事项 ID |
| data.list[].reasonName | String | 事项名称 |
| data.list[].enabled | boolean | 是否启用 |
| data.list[].defaultPoints | int | 默认积分值 |
| data.maxPoints | int | 单次操作最大积分限制 |
| requestId | String | 请求唯一标识 |

## 业务逻辑

1. `direction` 为必填查询参数，可选值为 `add`（增加）或 `deduct`（消耗）。
2. 不同 `direction` 返回不同的事项列表：增加积分事项如"活动赠送"、"借阅奖励"等；消耗积分事项如"积分兑换"、"积分抵扣"等。
3. `maxPoints` 表示单次操作允许的最大积分值（当前为 99999），前端应据此限制输入范围。
4. `memberId` 参数为预留，未来可根据会员等级或标签过滤可用事项。
5. `enabled` 为 `false` 的事项不应在前端展示或应置灰不可选。
6. `defaultPoints` 为选中该事项时的默认积分值，员工可在此基础上手动调整。
