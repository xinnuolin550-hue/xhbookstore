# 解析会员码

> **接口路径**: `POST /api/mp/v1/staff/member-code/scan`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` — `scanMemberCode()`

## 接口说明

员工通过微信小程序扫描会员出示的会员码，将扫码结果传递至后端解析，获取会员的唯一标识 `memberId`，用于后续查询会员信息、办理借阅等操作。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| scanResult | String | 是 | — | 扫码结果字符串，格式：`MEMBER:{memberNo}:TIMESTAMP:{timestamp}` |

### 字段说明

- `scanResult` 由会员小程序端生成的会员码二维码内容，前缀 `MEMBER:` 后跟会员编号，`TIMESTAMP:` 后跟生成时间的时间戳（毫秒）。
- 示例值：`MEMBER:65000000001:TIMESTAMP:1719748800000`

## 请求报文

```json
{
  "scanResult": "MEMBER:65000000001:TIMESTAMP:1719748800000"
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/member-code/scan \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{
    "scanResult": "MEMBER:65000000001:TIMESTAMP:1719748800000"
  }'
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功 |
| message | String | 响应消息 |
| data | object | 返回数据 |
| data.memberId | String | 解析得到的会员唯一 ID |
| requestId | String | 请求唯一标识 |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberId": "1"
  },
  "requestId": "req_002b3c4d"
}
```

## 错误响应

```json

  "code": 400,

  "data": null,
  "requestId": "req_002b3c4d"
}
```

  "data": null,
  "requestId": "req_002b3c4d"
}
```

### 会员码已过期（20005）

  "code": 20005,
  "message": "会员码已过期",
  "data": null,
  "requestId": "req_002b3c4d"

```

```json

  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_002b3c4d"

```

## 业务逻辑

1. 接收扫码结果字符串，提取 `scanResult` 参数，校验不为空。
2. 解析字符串中的 `memberNo`（会员编号）和 `timestamp`（生成时间戳）。
3. 校验会员码是否在有效期（例如生成时间距今不超过 60 秒），超时返回 `MEMBER_CODE_EXPIRED`。
4. 会员号在系统中不存在或已注销时返回 `MEMBER_CODE_INVALID`。
5. 校验通过后，根据 `memberNo` 查询会员真实 `memberId` 并返回。
6. 解析过程应做防伪校验（如签名校验），防止二维码被篡改。
