# 校验登录态

> **接口路径**: `GET /api/mp/v1/auth/session`
> **认证要求**: 是（需在 Header 中携带 `Authorization: Bearer <accessToken>`）
> **Content-Type**: 无（GET 请求）
> **对应代码**: `AuthController.java`

## 接口说明

校验当前 `accessToken` 是否有效，返回用户身份信息。适用于小程序启动时检查登录态是否过期，或页面权限校验。

---

## 请求参数

无请求体。需在请求头中携带认证信息。

### 请求头

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| Authorization | String | 是 | 无 | Bearer 认证，格式：`Bearer <accessToken>` |

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 提示信息 |
| data.valid | Boolean | 登录态是否有效（true=有效/false=无效） |
| data.isStaff | Boolean | 是否为已注册会员（valid=true 时有值） |
| data.userId | String | 用户 ID（valid=true 时有值） |
| requestId | String | 本次请求唯一标识 |

## 成功响应

### Token 有效（已注册会员）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": true,
    "userId": "123"
  },
  "requestId": "req_20260630120201"
}
```

### Token 有效（访客）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": false,
    "userId": "c3a7e8b0-1234-4e56-8901-abcdef012345"
  },
  "requestId": "req_20260630120202"
}
```

## 错误响应

### 未提供 Token 或 Token 无效（由拦截器拦截，未进入控制器）

```json
{
  "code": 10004,
  "message": "无效的访问令牌",
  "data": null,
  "requestId": "req_20260630120203"
}
```

### Token 已过期

```json
{
  "code": 10003,
  "message": "登录已过期，请重新登录",
  "data": null,
  "requestId": "req_20260630120204"
}
```

## 业务逻辑

1. 请求经 JWT 认证拦截器，从 `Authorization` 头中提取并解析 `accessToken`
2. 将解析得到的 `userId`、`isStaff` 等属性注入 `HttpServletRequest` 的属性中
3. 控制器从 `request.getAttribute("userId")` 取值，`null` 则 `valid=false`
4. 该接口**不校验 Token 是否已在服务端注销**（logout 仅标记，Token 本身仍可解析）

## 代码关联

| 文件 | 说明 |
|------|------|
| `AuthController.java` | 控制器，返回登录态信息 |
| JWT 拦截器（如 `JwtAuthenticationFilter`） | 解析 Token 并设置请求属性 |
