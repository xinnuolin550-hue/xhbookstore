# 退出登录

> **接口路径**: `POST /api/mp/v1/auth/logout`
> **认证要求**: 是（需在 Header 中携带 `Authorization: Bearer <accessToken>`）
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `AuthController.java`

## 接口说明

退出当前登录状态。当前实现为幂等操作，直接返回成功；后续版本可接入 Redis 黑名单机制使 Token 立即失效。

---

## 请求参数

无请求体。需在请求头中携带认证信息。

### 请求头

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| Authorization | String | 是 | 无 | Bearer 认证，格式：`Bearer <accessToken>` |

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 提示信息 |
| data.success | Boolean | 退出结果（始终为 true） |
| requestId | String | 本次请求唯一标识 |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "success": true
  },
  "requestId": "req_20260630120301"
}
```

## 错误响应

### 未提供 Token 或 Token 无效

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630120302"
}
```

### Token 已过期

```json
{
  "code": 10003,
  "message": "登录已过期，请重新登录",
  "data": null,
  "requestId": "req_20260630120303"
}
```

## 业务逻辑

1. 当前为轻量实现：不校验 Token 有效性，直接返回成功
2. **注意**：logout 操作不使已签发的 JWT 立即失效，客户端应自行清除本地存储的 Token
3. 后续优化方向：引入 Redis Token 黑名单机制，将 `accessToken` 的 `jti` 加入黑名单直到其自然过期
4. 前端应在调用 logout 后清除本地缓存的 `accessToken`、`refreshToken` 等信息，并跳转回登录页
