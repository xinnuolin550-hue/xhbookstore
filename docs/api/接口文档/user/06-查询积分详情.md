# 查询积分详情

> **接口路径**: `GET /api/mp/v1/user/points-records/{pointsRecordId}`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `UserController.java` — `pointsDetail()` 方法

## 接口说明

根据积分记录编号查询单条积分变动的详细信息。**注意：当前实现为占位状态，仅返回 `pointsRecordId`，完整字段待实现。**

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pointsRecordId | String | 是 | — | 积分记录编号，路径参数（如 `IN20260628120000123456`） |

## 请求报文

（路径参数，无请求体）

```
GET /api/mp/v1/user/points-records/IN20260628120000123456
```

## curl 测试

```bash
curl "http://localhost:8091/api/mp/v1/user/points-records/IN20260628120000123456" \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0 表示成功） |
| message | String | 提示信息 |
| requestId | String | 请求追踪 ID |
| data.pointsRecordId | String | 积分记录编号 |

> **待完善**：当前接口仅返回 `pointsRecordId`，后续应补充以下完整字段：

| 完整字段（规划） | 类型 | 说明 |
|------|------|------|
| data.pointsRecordId | String | 积分记录编号 |
| data.reasonName | String | 积分变动原因 |
| data.direction | String | 变动方向（`add` / `deduct`） |
| data.pointsDelta | Integer | 本次变动积分数 |
| data.beforePoints | Integer | 变动前积分余额 |
| data.afterPoints | Integer | 变动后积分余额 |
| data.operatedAt | Long | 变动时间（毫秒时间戳） |
| data.remark | String | 备注说明 |
| data.relatedOrderNo | String | 关联业务单号（如借阅单号、订单号等） |

## 错误响应

**未登录（401）**

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630_150000_053"
}
```

**积分记录不存在（404）**

```json
{
  "code": 404,
  "message": "资源不存在",
  "data": null,
  "requestId": "req_20260630_150000_054"
}
```

> 当前代码未校验 `pointsRecordId` 是否存在，直接返回传入的 ID。待完整实现后应增加数据库查询与 404 判断。

## 业务逻辑

1. 根据路径参数 `pointsRecordId` 查询 `PointsOrder` 表，获取该条积分记录的完整信息。
2. 当前接口为预留占位，仅返回传入的 `pointsRecordId`，未进行数据库查询。
3. 后续实现应补充：查询 `PointsOrder` 表、校验记录归属于当前登录用户、返回完整字段。
4. 编号前缀规则：`IN` 开头为收入（增加），`EX` 开头为支出（扣除）。
