# 查询本人借阅记录

> **接口路径**: `GET /api/mp/v1/user/borrows`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `UserController.java` — `myBorrows()` 方法

## 接口说明

分页查询当前登录用户的借阅记录列表，包含借阅单号、图书明细、借阅状态等信息。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNo | int | 否 | 1 | 页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页条数 |

## 请求报文

（Query 参数，无请求体）

```
GET /api/mp/v1/user/borrows?pageNo=1&pageSize=20
```

## curl 测试

```bash
curl "http://localhost:8091/api/mp/v1/user/borrows?pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0 表示成功） |
| message | String | 提示信息 |
| requestId | String | 请求追踪 ID |
| data.memberDisplay | String | 会员脱敏显示（手机号） |
| data.yearBorrowCount | int | 年度累计借阅次数 |
| data.currentBorrowingCount | int | 当前正在借阅中的数量（未完结的订单数） |
| data.page | Object | 分页对象 |
| data.page.list | Array | 借阅记录列表 |
| data.page.list[].orderNo | String | 借阅单号（如 `DY20260630143000123456`） |
| data.page.list[].totalBookCount | Integer | 借阅图书总册数 |
| data.page.list[].borrowStatus | Integer | 借阅状态（0-待取书, 1-借阅中, 2-已归还, 3-部分归还, 4-逾期, 5-已取消） |
| data.page.list[].borrowTime | Long | 借阅时间（毫秒时间戳） |
| data.page.list[].details | Array | 借阅明细列表 |
| data.page.list[].details[].bookId | Long | 图书 ID |
| data.page.list[].details[].bookTitle | String | 图书名称 |
| data.page.list[].details[].bookIsbn | String | 图书 ISBN |
| data.page.list[].details[].borrowStatus | Integer | 该册借阅状态（0-借阅中, 1-已归还） |
| data.page.list[].details[].borrowTime | Long | 借出时间（毫秒时间戳） |
| data.page.list[].details[].dueDate | Long | 应还时间（毫秒时间戳） |
| data.page.list[].details[].returnTime | Long | 实际归还时间（毫秒时间戳，未还时为 null） |
| data.page.pageNo | int | 当前页码 |
| data.page.pageSize | int | 每页条数 |
| data.page.total | long | 总记录数 |
| data.page.hasMore | boolean | 是否还有更多数据 |

### 借阅状态枚举（borrowStatus）

| 值 | 说明 |
|----|------|
| 0 | 待取书 — 订单已创建，等待到店取书 |
| 1 | 借阅中 — 图书已取出，正在阅读 |
| 2 | 已归还 — 所有图书已归还，订单完结 |
| 3 | 部分归还 — 部分图书已还，部分仍在借 |
| 4 | 逾期 — 超过应还日期仍未归还 |
| 5 | 已取消 — 订单已取消 |

## 业务逻辑

1. 从 Token 中解析当前用户的 `memberId`，查询该会员的所有借阅订单。
2. 返回的 `details` 中 `bookTitle` 和 `bookIsbn` 需关联图书信息表查询，当前实现直接透传 `BookBorrowDetail` 中的 `bookName` 字段。
3. `currentBorrowingCount` 统计 `borrowStatus` 不等于 2（已归还）且不等于 5（已取消）的订单数量。
4. 分页逻辑：`hasMore = pageNo * pageSize < total`，用于前端判断是否还有下一页。
5. `dueDate`（应还时间）需根据借阅日期和会员卡类型计算，当前实现需后端补充。
