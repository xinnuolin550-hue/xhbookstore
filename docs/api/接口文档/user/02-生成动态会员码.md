# 生成动态会员码

> **接口路径**: `POST /api/mp/v1/user/member-code`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `UserController.java` — `generateMemberCode()` 方法

## 接口说明

生成一个 30 秒有效期的动态会员二维码内容，用于前台扫码验证会员身份。每次调用均生成新的唯一 `codeId` 与 `codeContent`。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| （空对象） | — | — | — | 请求体为空对象 `{}`，用户信息从 Token 解析 |

## 请求报文

```json
{}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/user/member-code \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d "{}"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0 表示成功） |
| message | String | 提示信息 |
| requestId | String | 请求追踪 ID |
| data.memberNo | String | 会员编号（如 `65000000001`） |
| data.codeId | String | 本次生成的动态码唯一标识（UUID） |
| data.codeContent | String | 会员码完整内容，格式：`MEMBER:{memberNo}:TIMESTAMP:{timestamp}` |
| data.expiresAt | Long | 过期时间（毫秒时间戳，当前时间 + 30s） |
| data.ttlSeconds | Integer | 有效时长（秒），固定 30s |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberNo": "65000000001",
    "codeId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "codeContent": "MEMBER:65000000001:TIMESTAMP:1759270500000",
    "expiresAt": 1759270530000,
    "ttlSeconds": 30
  },
  "requestId": "req_20260630_150000_011"
}
```

## 错误响应

**未登录（401）**

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630_150000_012"
}
```

**会员不存在（20001）**

```json
{
  "code": 20001,
  "message": "会员不存在",
  "data": null,
  "requestId": "req_20260630_150000_013"
}
```

**会员未办理会员卡，无法生成会员码（20004）**

```json
{
  "code": 20004,
  "message": "未办理会员卡",
  "data": null,
  "requestId": "req_20260630_150000_014"
}
```

## 业务逻辑

1. 每次请求生成全新的动态码，旧的动态码自动失效。
2. `codeId` 使用 UUID 随机生成，用于后端校验时查找对应的会员码记录。
3. `codeContent` 格式固定为 `MEMBER:{memberNo}:TIMESTAMP:{毫秒时间戳}`，前台将其编码为二维码展示。
4. 有效期固定为 30 秒（`ttlSeconds: 30`），超时后该会员码失效，扫码时将返回 `20005` 错误码。
5. 当前实现返回固定 memberNo `65000000001`，待接入认证模块后从 Token 获取真实会员编号。
