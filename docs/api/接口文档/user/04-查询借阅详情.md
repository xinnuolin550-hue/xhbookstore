# 查询借阅详情

> **接口路径**: `GET /api/mp/v1/user/borrows/{borrowId}`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `UserController.java` — `borrowDetail()` 方法

## 接口说明

根据借阅单号查询单笔借阅订单的完整信息，包括订单概要、每本图书的借阅明细、以及归还记录。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| borrowId | String | 是 | — | 借阅单号，路径参数（如 `DY20260630143000123456`） |

## 请求报文

（路径参数，无请求体）

```
GET /api/mp/v1/user/borrows/DY20260630143000123456
```

## curl 测试

```bash
curl "http://localhost:8091/api/mp/v1/user/borrows/DY20260630143000123456" \
  -H "Authorization: Bearer <accessToken>"
```

## 错误响应

**借阅单号不存在（404）**

```json
{
  "code": 404,
  "message": "资源不存在",
  "data": null,
  "requestId": "req_20260630_150000_033"
}
```

> 当前代码中 `BookBorrowOrder order = bookBorrowService.selectOrderByNo(borrowId)` 返回 null 时返回空 `data`（`data: {}`），后续建议改为返回 404 错误码。

**未登录（401）**

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630_150000_034"
}
```

## 业务逻辑

1. 通过 `borrowId`（借阅单号）查询 `BookBorrowOrder`，若订单不存在则返回空对象或 404。
2. 根据订单 ID 查询关联的借阅明细列表（`BookBorrowDetail`）。
3. 根据订单 ID 查询关联的归还记录列表（`BookReturnDetail`）。
4. `details` 数组中 `borrowStatus` 字段：0 表示该册仍在借阅中，1 表示已归还。
5. `returns` 数组空表示该订单还未有任何归还操作。
6. 当前实现未校验该借阅单是否属于当前登录用户，后续应补充用户权限校验。
