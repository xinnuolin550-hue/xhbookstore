# 查询本人积分记录

> **接口路径**: `GET /api/mp/v1/user/points-records`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `UserController.java` — `myPointsRecords()` 方法

## 接口说明

分页查询当前登录用户的积分变动记录，包含积分增减明细、变动前后的余额等信息。

## 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNo | int | 否 | 1 | 页码，从 1 开始 |
| pageSize | int | 否 | 20 | 每页条数 |

## 请求报文

（Query 参数，无请求体）

```
GET /api/mp/v1/user/points-records?pageNo=1&pageSize=20
```

## curl 测试

```bash
curl "http://localhost:8091/api/mp/v1/user/points-records?pageNo=1&pageSize=20" \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0 表示成功） |
| message | String | 提示信息 |
| requestId | String | 请求追踪 ID |
| data.memberDisplay | String | 会员脱敏显示（手机号） |
| data.currentPoints | Integer | 当前可用积分余额 |
| data.yearEarnedPoints | Integer | 本年度已获得积分（仅统计增加） |
| data.page | Object | 分页对象 |
| data.page.list | Array | 积分记录列表 |
| data.page.list[].pointsRecordId | String | 积分记录编号（如 `IN20260628120000123456`） |
| data.page.list[].reasonName | String | 积分变动原因（如"借书获得"、"购书获得"、"兑换商品"等） |
| data.page.list[].direction | String | 变动方向：`add` 增加 / `deduct` 扣除 |
| data.page.list[].pointsDelta | Integer | 本次变动积分数（正数） |
| data.page.list[].beforePoints | Integer | 变动前积分余额 |
| data.page.list[].afterPoints | Integer | 变动后积分余额 |
| data.page.list[].operatedAt | Long | 变动时间（毫秒时间戳） |
| data.page.pageNo | int | 当前页码 |
| data.page.pageSize | int | 每页条数 |
| data.page.total | long | 总记录数 |
| data.page.hasMore | boolean | 是否还有更多数据 |

## 业务逻辑

1. 从 Token 中解析当前用户的 `memberId`，查询该会员的所有积分变动记录。
2. `pointsRecordId` 格式说明：以 `IN` 开头表示收入（增加），以 `EX` 开头表示支出（扣除）。当前代码根据 `orderNumber` 是否以 `IN` 开头判断 `direction`。
3. `direction` 为 `add` 时 `pointsDelta` 为正数；`deduct` 时 `pointsDelta` 为正数（表示扣除的绝对值）。
4. `beforePoints` 和 `afterPoints` 展示变动前后的积分余额，便于用户核对。
5. `yearEarnedPoints` 统计本年度所有 `direction=add` 的积分总和。
6. 当前实现返回固定值 `memberDisplay`、`currentPoints` 和 `yearEarnedPoints`，待接入认证模块后从真实数据计算。
