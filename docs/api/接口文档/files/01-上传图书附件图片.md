# 上传图书附件图片

> **接口路径**: `POST /api/mp/v1/files/book-attachment-images`
> **认证要求**: 是（需在 Header 中携带 `Authorization: Bearer <accessToken>`）
> **Content-Type**: multipart/form-data
> **对应代码**: `FileController.java`

## 接口说明

上传借书/还书时拍摄的图书附件图片，上传至腾讯云 COS 对象存储。支持三种业务场景：借书拍摄（imageType=1）、还书拍摄（imageType=2）、损坏记录（imageType=3）。图片限制为 10MB 以内，仅支持 image/* 类型。

根据传入的关联业务参数，系统会自动将图片记录写入对应的数据库表（借书明细图片表或图书图片表）。

---

## 请求参数

### 请求头

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| Authorization | String | 是 | 无 | Bearer 认证，格式：`Bearer <accessToken>` |

### 表单字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| file | File | 是 | 无 | 图片文件，最大 10MB，仅支持 image/* 类型（jpg/png/webp 等） |
| memberId | String | 是 | 无 | 会员 ID，用于构建 COS 存储路径 `bookstore/{memberId}/` |
| borrowOrderNo | String | 否 | 无 | 借书单号，关联借书单时传入 |
| borrowDetailId | Long | 否 | 无 | 借书明细 ID，传入后自动写入 `book_borrow_detail_image` 表 |
| borrowOrderId | Long | 否 | 无 | 借书单 ID，随明细一起写入 |
| imageType | Integer | 否 | 1 | 图片类型：1-借书拍摄 2-还书拍摄 3-损坏记录 |
| bookId | Long | 否 | 无 | 图书 ID，传入后自动写入 `book_image` 表 |

> **注意**: `borrowDetailId` + `borrowOrderNo` 同时传入时写入借书明细图片表；`bookId` 传入时写入图书图片表；两者均不传入时仅上传至 COS 不做数据库记录，由业务方后续调用关联。

| code | Integer | 0=成功 |

| data.imageId | String | COS 生成的图片唯一标识 ID |

## 业务逻辑

1. 校验图片文件：非空检查 -> 大小检查（上限 10MB）-> MIME 类型检查（必须以 `image/` 开头）
2. 上传至腾讯云 COS，存储路径格式：`bookstore/{memberId}/{yyyyMMdd}/{uuid}.{ext}`
3. COS 返回 `imageId`（存储路径）、`url`（原图访问地址）、`thumbUrl`（自动生成的缩略图）
4. 根据关联参数决定写入哪张图片表：
   - `borrowDetailId + borrowOrderNo` 同时存在 -> 写入 `book_borrow_detail_image` 表
   - `bookId` 存在 -> 写入 `book_image` 表
   - 两者都不存在 -> 仅上传，不做数据库记录（由业务方后续手动关联）
5. 上传操作人（staffId）从 Token 中解析，默认值 `"system"`
6. COS 上传异常时统一捕获为 `FILE_UPLOAD_FAILED`（50001），携带原始异常信息
