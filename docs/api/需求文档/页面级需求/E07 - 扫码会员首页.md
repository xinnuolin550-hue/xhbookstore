# E07 — 扫码会员首页

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff`

员工扫码或搜索会员后进入的会员概要页面，展示会员基本信息、卡信息以及当前可操作的业务权限状态（借阅、还书、积分操作等）。

|------|------|------|------|
| 扫码会员首页 | GET | `/api/mp/v1/staff/members/{memberId}/overview` | 是 |

## 1. 扫码会员首页

```
GET /api/mp/v1/staff/members/{memberId}/overview
Authorization: Bearer <accessToken>
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| memberId | String | 是 | 会员ID（路径参数） |

## 业务逻辑

1. **会员查询**: 根据 memberId 查询会员主表信息。
2. **手机号脱敏**: 手机号中间4位显示为 `****`。
3. **会员卡信息组装**:
   - cardStatus 根据会员 `status` 字段和 `validDate` 计算
   - remainingDays = (expiredAt - now) / 86400000
   - 卡等级由 `levelId * 10` 计算（1=普通卡(10) 2=银卡(20) 3=金卡(30) 4=钻石卡(40)）
4. **操作权限计算**:
   - `canBorrow`: 会员卡有效（未过期、未冻结、未挂失）且剩余借阅额度充足
   - `canReturn`: 会员有未还清的借阅记录
   - `canAddPoints`: 会员卡有效
   - `canDeductPoints`: 会员卡有效且当前积分 > 0
   - `maxAddPoints`: 不超过系统配置的单次上限（默认 99999）
   - `maxDeductPoints`: 不超过当前可用积分
5. **借阅统计**:
   - currentBorrowingCount: 统计状态为"借阅中"(0)和"部分归还"(1)的订单数
   - yearBorrowCount: 统计当年1月1日至今的所有借阅数量
