# C05 — 通用确认弹窗

> 对应代码: 前端通用组件 `ConfirmDialog` | 无独立 API

通用确认弹窗组件，用于所有需要用户二次确认的操作场景，包括但不限于注销账号、还书确认、积分操作确认等。

---

## 组件概述

确认弹窗是贯穿全应用的基础 UI 组件，在各业务页面中复用。弹窗由前端触发，本身不调用独立 API，但在用户确认后会触达对应的业务接口。

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| visible | Boolean | 是 | false | 弹窗显隐 |
| title | String | 是 | — | 弹窗标题 |
| message | String | 是 | — | 弹窗内容主体 |
| confirmText | String | 否 | "确定" | 确认按钮文字 |
| cancelText | String | 否 | "取消" | 取消按钮文字 |
| confirmType | String | 否 | "primary" | 确认按钮样式：primary / danger / warning |
| showCancel | Boolean | 否 | true | 是否显示取消按钮 |
| loading | Boolean | 否 | false | 确认中加载状态 |
| disabled | Boolean | 否 | false | 确认按钮禁用状态 |

### 组件事件

| 事件名 | 触发时机 | 回调参数 |
|--------|----------|----------|
| onConfirm | 用户点击确认按钮 | 无 |
| onCancel | 用户点击取消按钮或遮罩层 | 无 |
| onClose | 弹窗关闭（确认/取消/遮罩） | confirm: Boolean |

## 使用场景

### 场景 1：注销账号确认

**触发路径**: 设置页 → 注销须知页 → 确认注销

**弹窗参数**:
```
title: "确认注销账号"
message: "注销后您的会员权益将全部清空，历史借阅记录将被匿名化处理，此操作不可撤销。"
confirmText: "确认注销"
confirmType: "danger"
```

**确认后调用**: `POST /api/mp/v1/account/cancel`

**确认后响应（成功）**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "success": true,
    "cancelledAt": 1751234567890
  },
  "requestId": null
}
```

**确认后响应（失败）**:
```json
{
  "code": 60001,
  "message": "当前账号不允许注销",
  "data": null,
  "requestId": null
}
```

### 场景 2：还书确认

**触发路径**: 员工端 → 会员借阅详情页 → 点击"办理还书"

**弹窗参数**:
```

message: "即将为会员'张小明'归还以下图书：\n1. JavaScript高级程序设计\n2. 算法导论\n共 2 本。"
confirmText: "确认还书"
confirmType: "primary"
```

**确认后调用**: `POST /api/mp/v1/staff/borrow-returns`

**确认后响应（成功）**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "returnCount": 2,
    "returnOrderNo": "RT20260630143000123456"
  },
  "requestId": null
}
```

### 场景 3：积分扣减确认

**触发路径**: 员工端 → 会员详情 → 积分操作 → 扣减积分

**弹窗参数**:


## 业务逻辑

1. **二次确认**: 所有涉及数据变更（借书、还书、积分增减、注销）的操作前必须弹出确认弹窗。
2. **加载状态**: 用户点击确认后，弹窗按钮进入 loading 状态，防止重复提交。
3. **错误处理**: 接口返回失败时，弹窗保持开启并展示错误信息，用户可修改后重试。
4. **不可逆操作提醒**: 注销等不可逆操作，message 中须显式说明"此操作不可撤销"。
