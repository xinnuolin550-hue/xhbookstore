# C10 — 扫码登录确认页

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff/member-code/scan`

员工端通过扫描会员动态二维码获取会员信息的页面。扫码后解析会员码内容，加载会员概要信息供员工确认登录。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 扫描会员码 | POST | `/api/mp/v1/staff/member-code/scan` | 是 |
| 查询会员概要 | GET | `/api/mp/v1/staff/members/{memberId}/overview` | 是 |

---

## 扫码流程

```
员工打开扫码页
  │
  ├─→ 调用微信 wx.scanCode() 扫码
  │     │
  │     └─→ 得到 scanResult: "MEMBER:65000000001:TIMESTAMP:1751234567000"
  │
  ├─→ POST /api/mp/v1/staff/member-code/scan
  │     │
  │     └─→ 解析会员码 → 返回 memberId + memberNo
  │
  ├─→ GET /api/mp/v1/staff/members/{memberId}/overview
  │     │
  │     └─→ 加载会员概要信息
  │
  └─→ 展示会员确认页 → 员工确认后可进行借阅/积分等操作
```

## 1. 扫描会员码

```
POST /api/mp/v1/staff/member-code/scan
Authorization: Bearer <accessToken>
Content-Type: application/json;charset=UTF-8
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| scanResult | String | 是 | 微信扫一扫返回的二维码内容 |

### 请求报文

```json
{
  "scanResult": "MEMBER:65000000001:TIMESTAMP:1751234567000"
}
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.memberId | String | 会员 ID |
| data.memberNo | String | 会员卡号 |
| data.codeValid | Boolean | 会员码是否在有效期内 |
| data.expiresAt | Long | 会员码过期时间戳 |
| data.currentTimestamp | Long | 服务器当前时间戳 |

### 响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberId": "1",
    "memberNo": "65000000001",
    "codeValid": true,
    "expiresAt": 1751234597000,
    "currentTimestamp": 1751234568000
  },
  "requestId": null
}
```

### 错误响应（无效会员码）

```json
{
  "code": 20006,
  "message": "无效的会员码",
  "data": null,
  "requestId": null
}
```

### 错误响应（会员码已过期）

```json
{
  "code": 20005,
  "message": "会员码已过期",
  "data": null,
  "requestId": null
}
```

## 会员码格式

```
MEMBER:{memberNo}:TIMESTAMP:{timestamp}
```

| 段 | 说明 | 示例 |
|----|------|------|
| MEMBER | 固定前缀，标识为会员码 | MEMBER |
| memberNo | 会员卡号，8~11 位数字 | 65000000001 |
| TIMESTAMP | 固定分隔符 | TIMESTAMP |
| timestamp | 生成时间的毫秒时间戳 | 1751234567000 |

## 会员码有效期

- 动态二维码有效期 30 秒（由 `ttlSeconds` 字段控制）。
- 服务端在校验时检查 `TIMESTAMP` 与当前服务器时间之差是否小于 30 秒。
- 超时后返回 `code=20005`（MEMBER_CODE_EXPIRED），前端提示用户刷新二维码重新扫码。

## 页面布局

| 区域 | 内容 |
|------|------|
| 顶部 | 扫码取景框（全屏相机预览区域） |
| 扫码成功 | 会员头像 + 姓名 + 手机号 + 会员卡信息 |
| 操作区 | 借阅、还书、积分操作快捷入口（根据 availability 启禁） |
| 底部 | "取消"按钮，返回上一页 |

## 业务逻辑

1. **双阶段加载**: 扫码后先调 `scan` 接口解析会员码，再调 `overview` 接口加载详细数据。两步之间展示加载动画。
2. **可用性控制**: `availability` 字段由后端根据会员卡状态、当前借阅情况、员工权限综合计算，前端据此启禁相应操作按钮。
3. **会员码防伪**: 动态会员码包含时间戳，服务端校验有效期，防止截图重放攻击。
4. **扫码失败处理**: 无法解析的二维码内容给出"无效的会员码"提示，非会员码内容也给出明确提示。
