# U01 — 用户首页

> 对应代码: `UserController.java` | 基础路径: `/api/mp/v1/user`

用户进入小程序后的首页，展示会员核心信息概览，包括会员卡信息、可用积分、当前借阅数量和年度借阅统计。未注册会员时仅显示手机号脱敏展示，引导用户开通会员。

---

|------|------|------|------|
| 用户首页 | GET | `/api/mp/v1/user/home` | 是 |

## 1. 用户首页

```
GET /api/mp/v1/user/home
Authorization: Bearer <accessToken>
```

### 请求参数

无（用户身份从 Token 中解析）。

## 业务逻辑

1. **身份识别**: 从 JWT Token 中解析 userId 和 memberId，若 memberId 为 null 表示未开通会员。
2. **手机号脱敏**: 手机号中间4位统一显示为 `****`，脱敏规则与登录日志一致。
3. **会员信息组装**: 已注册会员需关联查询 Member 主表、会员扩展表、卡类型信息。
4. **借阅统计**: currentBorrowingCount 统计状态不为"已归还"(2)和"已取消"(5)的借阅单数量。
5. **年度统计**: yearBorrowCount 统计当年1月1日至今的所有借阅数量（含已归还）。
6. **缓存策略**: 首页数据可缓存 30 秒，减少高频查询对数据库的压力。
