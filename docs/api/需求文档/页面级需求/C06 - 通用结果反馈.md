# C06 — 通用结果反馈

> 对应代码: 前端通用组件 `ResultFeedback` | 无独立 API

通用结果反馈组件，用于在所有操作完成后向用户展示操作结果（成功/失败/警告）。本组件负责将后端 API 响应映射为前端用户可读的视觉反馈。

---

## 组件概述

结果反馈组件是贯穿全应用的 UI 组件，在 API 调用完成后自动或手动触发展示。组件根据 `code` 值自动匹配对应的样式模板。

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| visible | Boolean | 是 | false | 反馈显隐 |
| type | String | 是 | — | 反馈类型：success / error / warning / info |
| title | String | 是 | — | 反馈标题 |
| message | String | 否 | — | 反馈详情描述 |
| duration | Number | 否 | 2000 | 自动关闭时长（ms），0=不自动关闭 |
| showIcon | Boolean | 否 | true | 是否显示图标 |
| showClose | Boolean | 否 | true | 是否显示关闭按钮 |
| actionText | String | 否 | — | 操作按钮文字（如"查看详情"） |

### 组件事件

| 事件名 | 触发时机 | 回调参数 |
|--------|----------|----------|
| onAction | 用户点击操作按钮 | 无 |
| onClose | 反馈关闭 | 无 |

---

## API 响应码映射规则

| HTTP 状态码 | code | message | 反馈类型 | 反馈标题 | 用户提示 |
|------------|------|---------|----------|----------|----------|
| 200 | 0 | "操作成功" | success | 操作成功 | 展示成功图标 + message |
| 200 | ≥1（业务错误） | 如"积分不足" | error | 操作失败 | 展示错误图标 + 具体错误信息 |
| 400 | 400 | "参数无效" | error | 请求有误 | 请检查输入内容 |
| 401 | 401 | "未登录或登录已过期" | warning | 登录已过期 | 请重新登录 |
| 403 | 403 | "无权限访问" | warning | 权限不足 | 您没有该操作的权限 |
| 404 | 404 | "资源不存在" | error | 未找到 | 请求的资源不存在 |
| 405 | 405 | "请求方法不允许" | error | 请求错误 | 请稍后重试 |
| 500 | 500 | "服务器内部错误" | error | 服务器繁忙 | 请稍后重试 |

## 使用场景

### 场景 1：操作成功（积分增加）

**触发**: 员工端 → 积分增加成功

**展示参数**:
```
type: "success"
title: "积分增加成功"
message: "已为会员'张小明'增加 50 积分，当前积分 1750"
```

**对应后端响应**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "success": true,
    "pointsRecordId": "IN1751234567890",
    "beforePoints": 1700,
    "pointsDelta": 50,
    "afterPoints": 1750,
    "operatedAt": 1751234567890
  },
  "requestId": null
}
```

**UI 表现**: 绿色对号图标 + 标题 + 描述文字 + 3 秒自动关闭

### 场景 2：操作失败（积分不足）

**触发**: 员工端 → 积分扣减失败

**展示参数**:
```
type: "error"
title: "操作失败"
message: "积分不足，当前积分 100，需扣减 500"
```

**对应后端响应**:
```json
{
  "code": 30001,
  "message": "积分不足",
  "data": null,
  "requestId": "req_20260630_143020_020"
}
```

**UI 表现**: 红色叉号图标 + 标题 + 错误详情 + 手动关闭

### 场景 3：权限异常（Token 过期）

**触发**: 任意接口返回 401

**展示参数**:
```
type: "warning"
title: "登录已过期"
message: "请重新登录以继续使用"
actionText: "重新登录"
```

**对应后端响应**:
```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630_143021_021"
}
```

**UI 表现**: 黄色警告图标 + 标题 + 描述 + "重新登录"按钮点击跳转登录页

### 场景 4：服务器错误

**触发**: 任意接口返回 500

**展示参数**:
```
type: "error"
title: "服务器繁忙"
message: "当前服务暂不可用，请稍后重试"
```

**对应后端响应**:
```json
{
  "code": 500,
  "message": "服务器内部错误",
  "data": null,
  "requestId": "req_20260630_143022_022"
}
```

**UI 表现**: 红色叉号图标 + 标题 + 描述 + 手动关闭

### 场景 5：自定义成功提示

**触发**: 还书操作成功，但需展示详细结果

**展示参数**:
```
type: "success"
title: "还书成功"
message: "已归还 2 本图书，其中 1 本有轻微污损已记录"
actionText: "查看详情"
```

**对应后端响应**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "returnCount": 2,
    "returnOrderNo": "RT20260630143000123456",
    "damagedCount": 1,
    "damageNotes": "《JavaScript高级程序设计》封面有轻微污损"
  },
  "requestId": null
}
```

**UI 表现**: 绿色对号图标 + 标题 + 描述 + "查看详情"按钮跳转还书记录

## 视觉规范

| 类型 | 图标 | 图标色 | 背景色 |
|------|------|--------|--------|
| success | ✓ 圆形 | `#52C41A` | `#F6FFED` |
| error | ✕ 圆形 | `#FF4D4F` | `#FFF2F0` |
| warning | ! 三角 | `#FAAD14` | `#FFFBE6` |
| info | i 圆形 | `#1890FF` | `#E6F7FF` |

| 元素 | 规格 |
|------|------|
| 图标尺寸 | 80rpx（全屏）/ 48rpx（轻提示） |
| 标题字号 | 32rpx，加粗 |
| 正文字号 | 28rpx，颜色 `#666666` |
| 内边距 | 40rpx |

## 业务逻辑

1. **自动匹配**: 前端 HTTP 拦截器统一捕获响应，根据 `code` 自动触发对应类型的反馈。
2. **差异化展示**:
   - 成功类（code=0）：轻提示（Toast）形式展示，2~3 秒自动关闭。
   - 业务错误类（code≥1）：模态展示，需用户手动关闭。
   - 系统异常类（code=401/403/500）：模态展示，附带引导操作（重新登录/联系客服）。
3. **请求 ID 关联**: 当展示错误反馈时，可在调试模式中显示 `requestId` 用于问题排查。
