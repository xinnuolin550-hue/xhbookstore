# C08 — 借阅详情页

> 对应代码: `UserController.java` / `StaffController.java` | 基础路径: `/api/mp/v1/user/borrows` / `/api/mp/v1/staff/borrows`

借阅详情页是用户端和员工端共享的页面，展示单笔借书单的完整信息，包括订单概要、图书明细和还书记录。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 | 使用方 |
|------|------|------|------|--------|
| 查询用户端借阅详情 | GET | `/api/mp/v1/user/borrows/{borrowId}` | 是 | 用户端 |
| 查询员工端借阅详情 | GET | `/api/mp/v1/staff/borrows/{borrowId}` | 是 | 员工端 |

---

## 借阅状态枚举

| borrowStatus | 含义 | 说明 |
|-------------|------|------|
| 0 | 待借出 | 订单已创建，图书尚未实际出库 |
| 1 | 借出中 | 图书已出库，正在借阅期 |
| 2 | 已全部归还 | 所有图书均已归还 |
| 3 | 部分归还 | 部分图书已归还，部分借出中 |
| 4 | 已逾期 | 超过应还日期仍未归还 |
| 5 | 已取消 | 订单已取消 |

## 借阅明细状态枚举

| details[].borrowStatus | 含义 |
|------------------------|------|
| 0 | 借出中 |
| 1 | 已归还 |
| 2 | 部分归还 |

## 归还状态枚举

| returns[].returnStatus | 含义 | 说明 |
|------------------------|------|------|
| normal | 正常归还 | 图书完好 |
| damaged | 有损坏 | 图书有污损/破损，需记录 |
| lost | 丢失 | 图书遗失，需赔付 |

---

## 业务逻辑

1. **用户端 vs 员工端**: 用户端返回脱敏数据（手机号隐藏中4位），员工端返回完整信息。
2. **借阅状态判定**: `borrowStatus` 由系统根据借还记录自动计算，明细全部归还则该册状态变为"已归还"。
3. **图片关联**: 借阅明细详情中可能包含借书/还书时拍摄的图片（通过 `borrowDetailId` 关联 `book_borrow_detail_image` 表），前端在详情页底部展示图片列表。
4. **逾期判定**: 前端根据 `dueDate` 与当前时间比较，在界面上标记逾期图书。
