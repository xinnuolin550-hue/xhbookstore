# E10 — 积分操作页

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff`

员工为指定会员办理积分增加或消耗操作。先查询可用的积分事项（原因），再提交积分变动请求，系统通过悲观锁保证原子性。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 查询积分事项 | GET | `/api/mp/v1/staff/points-reasons` | 是 |
| 增加积分 | POST | `/api/mp/v1/staff/members/{memberId}/points/add` | 是 |
| 消耗积分 | POST | `/api/mp/v1/staff/members/{memberId}/points/deduct` | 是 |

## 1. 查询积分事项

```

Authorization: Bearer <accessToken>
```

## 2. 增加积分

```
POST /api/mp/v1/staff/members/{memberId}/points/add
Content-Type: application/json
Authorization: Bearer <accessToken>
```

> **注意**: 此接口含悲观锁 + 事务保证原子性，并发安全。

### 请求参数

**路径参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| memberId | String | 是 | 会员ID |

**请求体参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reasonId | String | 是 | 积分事项ID（须为增加类型） |
| points | Integer | 是 | 增加积分数值（正整数） |
| remark | String | 否 | 操作备注 |

### 请求报文示例

```json
{
  "reasonId": "1",
  "points": 50,
  "remark": "周年庆活动积分赠送"
}
```

```bash
curl -X POST "https://api.xhbookstore.com/api/mp/v1/staff/members/1/points/add" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "reasonId": "1",
    "points": 50,
    "remark": "周年庆活动积分赠送"
  }'
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 操作成功 |
| data.success | Boolean | 是否成功 |
| data.pointsRecordId | String | 积分记录编号（订单号） |
| data.beforePoints | Integer | 操作前积分 |
| data.pointsDelta | Integer | 本次变动积分值 |
| data.afterPoints | Integer | 操作后积分 |
| data.operatedAt | Long | 操作时间（毫秒时间戳） |

## 业务逻辑

### 增加积分 (addPoints)

1. **参数校验**:
   - reasonId 不能为空，且必须在积分事项表中存在且启用
   - points 不能为空，且必须为正整数
   - points 不能超过系统配置的单次最大积分（maxPoints）
2. **会员校验**: 会员必须存在且会员卡状态有效。
3. **悲观锁处理**: `SELECT ... FOR UPDATE` 锁定会员行记录，防止并发操作。
4. **积分计算**: afterPoints = beforePoints + points。
5. **记录生成**: 生成积分订单（订单号前缀 `IN`），记录变动前后的积分值和操作人。
6. **事务保证**: 更新会员积分 + 插入积分订单在同一事务中。

### 消耗积分 (deductPoints)

1. **参数校验**: 与增加积分相同。
2. **余额检查**: 校验 `member.currentPoints >= points`，不足则返回 `POINTS_NOT_ENOUGH(30001)`。
3. **记录生成**: 生成积分订单（订单号前缀 `OT`），pointsDelta 返回负值。
4. **TODO 事项**: 当前 deductPoints 为部分实现，扣减逻辑和出账单待完善。

### 积分事项 (pointsReasons)

- 根据 direction 参数返回对应方向的启用事项列表
- `add` 方向返回增加积分的事项（如活动赠送、借阅奖励等）
- `deduct` 方向返回消耗积分的事项（如兑换商品、积分抵扣等）
- 每个事项有可选的 defaultPoints，前端可作为输入框的默认值
