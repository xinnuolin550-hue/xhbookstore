# C11 — 组件预览页

> 对应代码: 前端预览页面 `pages/dev/component-preview/index` | 无独立 API

开发者组件预览页面，用于集中展示和调试项目中所有可复用的 UI 组件。仅开发环境可见，生产环境需通过配置隐藏入口。

---

## 页面概述

组件预览页列出了项目中的所有通用 UI 组件，每个组件展示其不同状态和用法。开发人员在该页面可快速查看组件的 props、slots、events，并交互式测试组件行为。

### 访问方式

- **开发环境**: 通过小程序开发工具直接跳转 `/pages/dev/component-preview/index`
- **生产环境**: 入口默认隐藏，可通过开启调试模式（连续点击版本号 5 次）显示

## 组件详述

### 1. ConfirmDialog（确认弹窗）

**对应文档**: [C05 - 通用确认弹窗](./C05%20-%20通用确认弹窗.md)

**Props**:

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| visible | Boolean | false | 弹窗显隐 |
| title | String | "提示" | 弹窗标题 |
| message | String | "" | 弹窗内容 |
| confirmText | String | "确定" | 确认按钮文字 |
| cancelText | String | "取消" | 取消按钮文字 |
| confirmType | String | "primary" | 按钮样式：primary / danger / warning |
| showCancel | Boolean | true | 是否显示取消按钮 |
| loading | Boolean | false | 确认按钮加载状态 |
| disabled | Boolean | false | 确认按钮禁用 |
| zIndex | Number | 1000 | 弹窗层级 |

**Events**:

| 事件 | 参数 | 说明 |
|------|------|------|
| confirm | — | 点击确认 |
| cancel | — | 点击取消 |
| close | `{ confirm: Boolean }` | 弹窗关闭 |

**预览页演示内容**:
- 基础弹窗（仅标题+内容）
- 危险操作弹窗（confirmType=danger）
- 加载中状态
- 长内容滚动

**代码示例**:
```html
<ConfirmDialog
  :visible="showDialog"
  title="确认操作"
  message="确认执行此操作吗？"
  confirmType="primary"
  @confirm="handleConfirm"
  @cancel="handleCancel"
/>
```

### 2. ResultFeedback（结果反馈）

**对应文档**: [C06 - 通用结果反馈](./C06%20-%20通用结果反馈.md)

**Props**:

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| visible | Boolean | false | 反馈显隐 |
| type | String | "success" | 类型：success / error / warning / info |
| title | String | "" | 反馈标题 |
| message | String | "" | 反馈详情 |
| duration | Number | 2000 | 自动关闭时长（0=不自动关闭） |
| showIcon | Boolean | true | 显示图标 |
| showClose | Boolean | true | 显示关闭按钮 |
| actionText | String | "" | 操作按钮文字 |

**Events**:

| 事件 | 参数 | 说明 |
|------|------|------|
| action | — | 点击操作按钮 |
| close | — | 反馈关闭 |

**预览页演示内容**:
- 成功反馈（绿色 + 对号）
- 错误反馈（红色 + 叉号）
- 警告反馈（黄色 + 感叹号）
- 信息反馈（蓝色 + 字母 i）
- 带操作按钮的结果反馈

**代码示例**:
```html
<ResultFeedback
  :visible="showResult"
  type="success"
  title="操作成功"
  message="已成功处理请求"
  :duration="3000"
  @close="showResult = false"
/>
```

### 8. MemberCard（会员卡片）

**Props**:

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| member | Object | — | 会员数据 |
| card | Object | — | 会员卡数据 |
| compact | Boolean | false | 紧凑模式（用于列表） |
| showActions | Boolean | true | 显示操作按钮 |

**member 对象结构**:
```json
{
  "memberId": "1",
  "memberNo": "65000000001",
  "memberName": "张小明",
  "phoneDisplay": "138****0001",
  "currentPoints": 1700
}
```

**card 对象结构**:
```json
{
  "cardName": "年度借阅卡",
  "cardStatus": "active",
  "remainingDays": 180,
  "memberLevel": 10
}
```

**Events**:

| 事件 | 参数 | 说明 |
|------|------|------|
| click | `{ memberId }` | 点击卡片 |
| borrow | `{ memberId }` | 点击借阅按钮 |
| return | `{ memberId }` | 点击还书按钮 |
| points | `{ memberId }` | 点击积分按钮 |

**预览页演示内容**:
- 标准会员卡片（含操作按钮）
- 紧凑模式卡片（列表用）
- 已过期会员卡（灰色样式）

**代码示例**:
```html
<MemberCard
  :member="memberData"
  :card="cardData"
  @borrow="handleBorrow"
  @return="handleReturn"
  @points="handlePoints"
/>
```

| 区域 | 内容 |

## 业务逻辑

1. **开发专用**: 本页面仅供开发/测试使用，生产环境通过 `ENV` 配置和调试模式控制入口显隐。
2. **组件注册**: 所有组件应统一在 `components/index.js` 中注册导出，预览页自动扫描注册列表进行展示。
3. **Props 面板**: 每个组件 Demo 下方展示 Props 控制面板，允许开发人员实时调整 Props 值查看组件响应。
4. **快照测试**: 预览页可用于组件的可视化回归测试，建议配合截图对比工具使用。
