# E08 — 办理借阅页

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff`

员工为已扫码确认身份的会员办理图书借阅业务。支持一次借阅多本图书，提交后生成借阅订单。

---

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 办理借阅 | POST | `/api/mp/v1/staff/members/{memberId}/borrows` | 是 |

---

## 1. 办理借阅

```
POST /api/mp/v1/staff/members/{memberId}/borrows
Content-Type: application/json
Authorization: Bearer <accessToken>
```

### 请求参数

**路径参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| memberId | String | 是 | 会员ID |

**请求体参数**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| books | Array | 是 | 借阅图书列表（至少1本） |
| books[].bookId | Long | 是 | 图书ID |
| books[].bookTitle | String | 是 | 图书名称 |
| books[].isbn | String | 否 | 图书ISBN |
| books[].borrowQty | Integer | 否 | 借阅数量，默认 1 |
| remark | String | 否 | 借阅备注 |

## 业务逻辑

1. **参数校验**:
   - 校验 memberId 对应会员是否存在
   - 校验 books 列表不能为空
   - 校验每本书的 bookId、bookTitle 不能为空
2. **会员资格校验**:
   - 会员卡是否有效（未过期、未冻结、未挂失）
   - 当前借阅数量 + 本次借阅数量 <= 会员卡允许的最大借阅数
3. **借阅额度校验**:
   - 查询会员的 borrowCountValid（有效借阅额度）
   - 比较当前已借阅数，判断额度是否充足
4. **生成借阅单**:
   - 订单号格式: `BO` + 日期(yyyyMMdd) + 4位序列号
   - 例如: `BO202606300001`
5. **创建借阅明细**: 每本图书生成一条借阅明细记录。
6. **应还时间计算**: 借阅时间 + 借阅期限（根据卡类型决定，通常为 30 天）。
7. **操作人**: 从 JWT Token 中获取当前员工信息。
