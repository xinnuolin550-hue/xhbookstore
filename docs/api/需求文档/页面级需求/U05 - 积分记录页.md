# U05 — 积分记录页

> 对应代码: `UserController.java` | 基础路径: `/api/mp/v1/user`

展示当前用户的积分（书城币）变动记录，支持分页查询。顶部展示会员标识、当前可用积分和年度累计获得积分。每条记录包含积分变动原因、增减方向、变动数量、变动前后余额及操作时间。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 我的积分记录（分页） | GET | `/api/mp/v1/user/points-records` | 是 |
| 积分详情 | GET | `/api/mp/v1/user/points-records/{pointsRecordId}` | 是 |

## 业务逻辑

1. **分页查询**: 按操作时间倒序排列，最新的积分变动排在前面。
2. **方向标识**: direction 字段根据订单号前缀判断——以 `IN` 开头为增加(`add`)，其余为扣除(`deduct`)。
3. **年度统计**: yearEarnedPoints 统计当年1月1日至今所有 direction=add 的 pointsDelta 之和。
4. **积分来源**: 系统在以下场景自动产生积分记录：
   - 借书归还后奖励积分（借书奖励）
   - 在书店购书赠送积分（购书赠送）
   - 每日签到奖励（签到奖励）
   - 积分商城兑换商品扣除积分（兑换消费）
   - 每年年底过期积分扣除（过期扣除）
5. **一致性**: beforePoints + pointsDelta * (±1) = afterPoints，前端可以此校验数据完整性。
6. **空数据兜底**: 新会员或无积分变动记录时返回空列表。
