# U03 — 借阅记录页

> 对应代码: `UserController.java` | 基础路径: `/api/mp/v1/user`

展示当前用户的借阅记录列表，支持分页查询。顶部展示会员标识、年度借阅总数和当前借阅数。每条借阅记录包含借书单号、图书数量、借阅状态、借书时间及该单下的图书明细（图书信息、借阅状态、应还日期等）。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 我的借阅记录（分页） | GET | `/api/mp/v1/user/borrows` | 是 |
| 借阅详情 | GET | `/api/mp/v1/user/borrows/{borrowId}` | 是 |

## 业务逻辑

1. **分页查询**: 按借书时间倒序排列，最新的借阅记录排在前面。
2. **借阅状态计算**:
   - currentBorrowingCount = 状态不为"已归还"(2)且不为"已取消"(5)的借阅单数量。
   - 同一借阅单中部分归还时整体仍算"当前借阅中"。
3. **年度统计**: yearBorrowCount = 当年1月1日至今所有借阅数量（含已归还和已取消）。
4. **权限控制**: 借阅详情接口需校验当前 Token 所属 memberId 与借阅单的 memberId 一致，避免越权查询。
5. **超期提醒**: 前端可根据 details[].dueDate 与当前时间对比，超期的书显示"已逾期"标识。
6. **空数据兜底**: 新会员无借阅记录时返回空列表，不返回错误。
