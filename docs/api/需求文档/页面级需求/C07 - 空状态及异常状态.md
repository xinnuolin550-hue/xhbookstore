# C07 — 空状态及异常状态

> 对应代码: 前端通用组件 `EmptyState` / `ErrorState` / `NetworkError` | 无独立 API

空状态和异常状态组件，用于展示列表无数据、网络异常、服务器错误、登录失效等情况下的占位 UI。

---

## 组件概述

| 组件名 | 状态类型 | 触发条件 |
|--------|----------|----------|
| `EmptyState` | 空数据 | API 返回 data.list=[] 且 data.total=0 |
| `NetworkError` | 网络异常 | HTTP 请求无响应（超时/断网） |
| `ServerError` | 服务器错误 | API 返回 code=500 或 HTTP 5xx |
| `AuthError` | 认证失效 | API 返回 code=401 或 HTTP 401 |

---

## 空状态（EmptyState）

### 触发条件

- 列表类接口返回 `data.list` 为空数组且 `data.total` 为 0。
- 用户首次使用，尚未产生任何数据记录。

### 示例场景

**场景一：借阅记录为空**

```
GET /api/mp/v1/user/borrows?pageNo=1&pageSize=20
Authorization: Bearer <accessToken>
```

**响应报文**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberDisplay": "138****0001",
    "yearBorrowCount": 0,
    "currentBorrowingCount": 0,
    "page": {
      "list": [],
      "pageNo": 1,
      "pageSize": 20,
      "total": 0,
      "hasMore": false
    }
  },
  "requestId": null
}
```

**UI 描述**:
- 居中插图：书架/书本图标（灰色调）
- 标题："暂无借阅记录"
- 副标题："去书店挑选一本好书开始阅读吧"
- 操作按钮："浏览图书"（跳转书库）

**场景二：积分记录为空**

```
GET /api/mp/v1/user/points-records?pageNo=1&pageSize=20
Authorization: Bearer <accessToken>
```

**响应报文**:
```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberDisplay": "138****0001",
    "currentPoints": 0,
    "yearEarnedPoints": 0,
    "page": {
      "list": [],
      "pageNo": 1,
      "pageSize": 20,
      "total": 0,
      "hasMore": false
    }
  },
  "requestId": null
}
```

**UI 描述**:
- 居中插图：积分/金币图标（灰色调）
- 标题："暂无积分记录"
- 副标题："借阅图书或参与活动可获得积分"
- 无操作按钮（纯提示性空状态）

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| icon | String | 否 | "empty" | 插图类型：empty / search / comment / book |
| title | String | 是 | — | 空状态主标题 |
| description | String | 否 | — | 空状态副标题 |
| actionText | String | 否 | — | 操作按钮文字（不传则不显示按钮） |
| actionLink | String | 否 | — | 按钮点击跳转路径 |

### 视觉规范

| 元素 | 规格 |
|------|------|
| 插图 | 200rpx x 200rpx，居中 |
| 标题字号 | 30rpx，颜色 `#333333` |
| 副标题字号 | 26rpx，颜色 `#999999` |
| 上下间距 | 标题距插图 32rpx，副标题距标题 16rpx |

## 网络异常（NetworkError）

### 触发条件

- HTTP 请求无响应（超时 / 断网 / DNS 解析失败）。
- WebSocket 连接断开。
- HTTP 层错误（非 HTTP 响应，如 `ERR_CONNECTION_REFUSED`）。

### UI 描述

- 居中插图：WiFi/信号图标（断开状态，灰色调）
- 标题："网络连接异常"
- 副标题："请检查网络设置后重试"
- 操作按钮："重新加载"

### 典型错误场景

前端 HTTP 拦截器捕获 `timeout` / `network error` 时触发：

```
Error: timeout of 10000ms exceeded
Error: Network Error (net::ERR_CONNECTION_REFUSED)
```

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| retry | Function | 否 | — | 重试回调函数 |
| retryText | String | 否 | "重新加载" | 重试按钮文字 |

### 视觉规范

| 元素 | 规格 |
|------|------|
| 插图 | 200rpx x 200rpx，居中 |
| 标题字号 | 30rpx，颜色 `#333333` |
| 副标题字号 | 26rpx，颜色 `#999999` |
| 按钮 | 主色 `#07C160`，圆角 40rpx |

## 服务器错误（ServerError / 500）

### 触发条件

- API 返回 HTTP 500。
- API 返回 `code=500`、`code=1` 或其它系统级错误码。
- API 返回格式异常（无法解析的响应体）。

### 示例

```json
{
  "code": 500,
  "message": "服务器内部错误",
  "data": null,
  "requestId": "req_20260630_143030_030"
}
```

### UI 描述

- 居中插图：服务器/齿轮图标（灰色调）
- 标题："服务暂不可用"
- 副标题："服务器繁忙，请稍后重试"
- 操作按钮："刷新页面"

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| retry | Function | 否 | — | 重试回调函数 |
| retryText | String | 否 | "刷新页面" | 重试按钮文字 |
| requestId | String | 否 | — | 展示 requestId 用于反馈 |

### 视觉规范

| 元素 | 规格 |
|------|------|
| 插图 | 200rpx x 200rpx，居中 |
| 标题字号 | 30rpx，颜色 `#333333` |
| 副标题字号 | 26rpx，颜色 `#999999` |
| requestId | 20rpx，颜色 `#CCCCCC`，可选展示 |

## 认证失效（AuthError / 401）

### 触发条件

- API 返回 HTTP 401。
- API 返回 `code=401`（UNAUTHORIZED）。
- API 返回 `code=10003`（AUTH_TOKEN_EXPIRED）。
- API 返回 `code=10004`（AUTH_TOKEN_INVALID）。

### 示例

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260630_143031_031"
}
```

```json
{
  "code": 10003,
  "message": "登录已过期，请重新登录",
  "data": null,
  "requestId": "req_20260630_143032_032"
}
```

### UI 描述

- 居中插图：用户/锁头图标（灰色调）
- 标题："登录已过期"
- 副标题："请重新登录以继续使用"
- 操作按钮："重新登录"（跳转至微信授权登录）

### 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| loginUrl | String | 否 | "/pages/login/index" | 登录页路径 |
| loginText | String | 否 | "重新登录" | 登录按钮文字 |

### 视觉规范

| 元素 | 规格 |
|------|------|
| 插图 | 200rpx x 200rpx，居中 |
| 标题字号 | 30rpx，颜色 `#333333` |

## 状态切换流程

```
请求发起
  │
  ├─→ 网络异常 ──→ NetworkError ──→ 重试 ──→ 重新请求
  │
  ├─→ 401 ──→ AuthError ──→ 重新登录 ──→ 重新请求
  │
  ├─→ 500 ──→ ServerError ──→ 刷新 ──→ 重新请求
  │
  └─→ 200 OK
        │
        ├─→ code=0, list=[] ──→ EmptyState
        ├─→ code=0, list=[...] ──→ 正常内容
        ├─→ code≥1 ──→ 业务错误提示（见 C06 通用结果反馈）
        └─→ code=401 ──→ AuthError
```

## 业务逻辑

1. **优先级**: 网络异常 > 认证失效 > 服务器错误 > 空数据。同一页面同时满足多个条件时，优先级最高的状态优先生效。
2. **页面级 vs. 区块级**: 空状态可作用于整个页面或页面中的某个区块（如 Tab 切换）。网络异常和服务器错误通常作用于整个页面。
3. **自动重试**: 网络恢复（通过 `window.addEventListener('online', ...)`）后自动重新加载当前页面。
4. **Token 刷新**: 遇到 401 时，前端应先尝试通过 refreshToken 续期（调用 `/auth/refresh-token`），续期失败才展示 AuthError。
