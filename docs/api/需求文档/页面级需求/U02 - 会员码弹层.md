# U02 — 会员码弹层

> 对应代码: `UserController.java` | 基础路径: `/api/mp/v1/user`

用户在首页点击"会员码"按钮时弹出动态二维码弹层。二维码内容包含会员编号和时间戳，后台每次生成新码，有效期 30 秒，过期后需重新生成，确保离线核销场景的安全性。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 生成动态会员码 | POST | `/api/mp/v1/user/member-code` | 是 |

## 1. 生成动态会员码

```
POST /api/mp/v1/user/member-code
Authorization: Bearer <accessToken>
Content-Type: application/json;charset=UTF-8
```

### 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| （空请求体） | - | - | - | 用户身份从 Authorization Token 中解析 |

### 请求报文

```json
{}
```

### 请求报文示例

```bash
curl -X POST "https://api.xhbookstore.com/api/mp/v1/user/member-code" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIiwiaXNTdGFmZiI6ZmFsc2UsInBob25lIjoiMTM4MTIzNDAwMDEiLCJtZW1iZXJJZCI6MSwidHlwZSI6ImFjY2VzcyIsImlhdCI6MTc1MTIzNDU2NywiZXhwIjoxNzUxMjQxNzY3fQ.xxxxx" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{}'
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 操作成功 |
| data.memberNo | String | 会员编号 |
| data.codeId | String | 本次生成码的唯一标识（UUID） |
| data.codeContent | String | 二维码编码内容，格式 `MEMBER:{memberNo}:TIMESTAMP:{timestamp}` |
| data.expiresAt | Long | 过期时间（毫秒时间戳），当前时间 + 30s |
| data.ttlSeconds | Integer | 有效期（秒），固定 30 |
| requestId | String | 请求追踪ID |

## 业务逻辑

1. **身份校验**: 从 Token 中解析 userId，根据 userId 查询关联的 memberId。若未开通会员则返回错误码 10001。
2. **动态生成**: 每次请求都生成新的唯一 codeId（UUID），codeContent 格式为 `MEMBER:{memberNo}:TIMESTAMP:{当前毫秒时间戳}`。
3. **有效期控制**: ttlSeconds 固定为 30 秒，expiresAt = 服务端当前时间 + 30000ms。前端应在此时间内完成核销。
4. **核销流程**: 收银员扫描会员码后解析 codeContent，校验会员号与时间戳。时间戳超过 30 秒应提示"二维码已过期，请重新出示"。
5. **单次有效性**: 该接口设计为每次生产新码，同一会员连续请求会生成不同的 codeId。建议前端配合防抖，避免用户无意识频繁生成。
6. **安全考量**: 二维码内容不含敏感身份信息（仅会员编号和时间戳），核销时后端需结合当前会话二次校验。
