# C09 — 积分详情页

> 对应代码: `UserController.java` / `StaffController.java` | 基础路径: `/api/mp/v1/user/points-records` / `/api/mp/v1/staff/points-records`

积分详情页展示单条积分记录的完整信息，包括积分变动前后的余额、变动原因和操作人信息。用户端和员工端共享此页面。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 | 使用方 |
|------|------|------|------|--------|
| 查询用户端积分详情 | GET | `/api/mp/v1/user/points-records/{pointsRecordId}` | 是 | 用户端 |
| 查询员工端积分详情 | GET | `/api/mp/v1/staff/points-records/{pointsRecordId}` | 是 | 员工端 |

---

## 1. 查询用户端积分详情

```
GET /api/mp/v1/user/points-records/{pointsRecordId}
Authorization: Bearer <accessToken>
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pointsRecordId | String | 是 | 积分记录编号（路径参数） |

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.pointsRecordId | String | 积分记录编号 |
| data.direction | String | 变动方向：add=增加 deduct=扣减 |
| data.reasonName | String | 积分事项名称 |
| data.reasonId | String | 积分事项 ID |
| data.pointsDelta | Integer | 变动积分值（增加为正数，扣减为负数） |
| data.beforePoints | Integer | 变动前积分余额 |
| data.afterPoints | Integer | 变动后积分余额 |
| data.operatedAt | Long | 操作时间戳（毫秒） |
| data.remark | String | 备注说明 |
| data.memberDisplay | String | 会员显示（脱敏手机号） |
| data.staffName | String | 操作员工姓名（员工端操作时有值） |

## 积分记录编号规则

| 前缀 | 含义 | 示例 |
|------|------|------|
| `IN` | 积分增加（Income） | `IN20260630143000123456` |
| `OT` | 积分扣减（Outgo） | `OT20260629150000123457` |

格式：`{前缀}{yyyyMMddHHmmss}{8位随机数}`

## 页面布局

| 区域 | 内容 |
|------|------|
| 标题区 | 积分变动方向图标 + 变动积分值（+50 绿色 / -200 红色） |
| 状态区 | 变动后余额 | 当前总积分 |
| 详情区 | 事项名称、备注、操作时间、操作员工 |
| 会员信息区（员工端） | 会员姓名、会员卡号、手机号 |

## 视觉规范

| 元素 | 规格 |
|------|------|
| 增加积分 | 绿色 `#52C41A`，带 `+` 号前缀 |
| 扣减积分 | 红色 `#FF4D4F`，带 `-` 号前缀 |
| 积分余额 | 字号 48rpx，颜色 `#333333` |
| 详情项 | 左侧标签 28rpx 灰色，右侧数值 28rpx 深色 |

## 业务逻辑

1. **积分记录不可篡改**: 积分订单采用 IN/OUT 记账模式，每条记录标记变动前后的余额，形成可审计的积分流水。
2. **来源区分**: `operationDevice` 字段区分操作来源 — 员工通过小程序操作标记为"小程序"，后台管理员操作标记为"后台管理"。
3. **权限控制**: 用户端只能查看本人的积分记录；员工端可查看全市所有会员的积分记录。
4. **积分过期**: 积分有效期跟随会员卡有效期，页面顶部可展示"积分将于 xxxx-xx-xx 过期"提示。
