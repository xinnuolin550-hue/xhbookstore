# C03 — 注销须知页

> 对应代码: `AccountController.java` | 基础路径: `/api/mp/v1/account`

用户注销账号前的资格检查和注销确认页面。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 查询注销前置状态 | GET | `/api/mp/v1/account/cancel-eligibility` | 是 |
| 注销账号 | POST | `/api/mp/v1/account/cancel` | 是 |

---

## 1. 查询注销前置状态

```
GET /api/mp/v1/account/cancel-eligibility
Authorization: Bearer <accessToken>
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.canCancel | Boolean | 是否可注销 |
| data.blockedReason | String | 不可注销原因（null=可注销） |
| data.hasUnreturnedBooks | Boolean | 是否有未还图书 |
| data.isStaffActive | Boolean | 是否在职员工 |
| data.currentPoints | Integer | 当前剩余积分 |

### 响应报文（可注销）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "canCancel": true,
    "blockedReason": null,
    "hasUnreturnedBooks": false,
    "isStaffActive": false,
    "currentPoints": 100
  },
  "requestId": null
}
```

### 响应报文（不可注销 — 有未还图书）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "canCancel": false,
    "blockedReason": "您还有 3 本图书未归还，请先归还后再注销",
    "hasUnreturnedBooks": true,
    "isStaffActive": false,
    "currentPoints": 50
  },
  "requestId": null
}
```

### 响应报文（不可注销 — 在职员工）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "canCancel": false,
    "blockedReason": "员工账号需联系管理员注销",
    "hasUnreturnedBooks": false,
    "isStaffActive": true,
    "currentPoints": 200
  },
  "requestId": null
}
```

## 2. 注销账号

```
POST /api/mp/v1/account/cancel
Authorization: Bearer <accessToken>
Content-Type: application/json;charset=UTF-8
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reason | String | 否 | 注销原因（可选） |
| confirmRead | Boolean | 是 | 确认已阅读注销须知 |

### 请求报文

```json
{
  "reason": "不再需要使用",
  "confirmRead": true
}
```

### 响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "success": true,
    "cancelledAt": 1751234567890
  },
  "requestId": null
}
```

### 错误响应（不可注销）

```json
{
  "code": 400,
  "message": "当前账号不符合注销条件：有未归还图书",
  "data": null,
  "requestId": null
}
```

---

## 业务逻辑

1. **注销前置检查**: 先调 `cancel-eligibility` 检查资格，前台展示阻断原因。
2. **阻断条件**: 有未还图书、在职员工、有未完成订单。
3. **注销执行**: 逻辑删除，会员状态标记为注销。
