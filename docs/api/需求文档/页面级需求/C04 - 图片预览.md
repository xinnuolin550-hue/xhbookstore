## 1. 上传图书附件图片

```
POST /api/mp/v1/files/book-attachment-images
Authorization: Bearer <accessToken>
Content-Type: multipart/form-data
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 图片文件（最大10MB，仅限 image/*） |
| memberId | String | 是 | 会员ID |
| borrowOrderNo | String | 否 | 借书单号（关联借书单时传入） |
| borrowDetailId | Long | 否 | 借书明细ID |
| borrowOrderId | Long | 否 | 借书单ID |
| imageType | Integer | 否 | 图片类型：1=借书拍摄 2=还书拍摄 3=损坏记录（默认1） |
| bookId | Long | 否 | 图书ID（上传封面时传入） |

### 请求报文（curl）

```bash
curl -X POST http://localhost:8091/api/mp/v1/files/book-attachment-images \
  -H "Authorization: Bearer <accessToken>" \
  -F "file=@book_photo.jpg" \
  -F "memberId=1" \
  -F "borrowOrderNo=DY20260630143000123456" \
  -F "borrowDetailId=1001" \
  -F "borrowOrderId=5001" \
  -F "imageType=1"
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.imageId | String | 图片唯一标识 |
| data.url | String | 图片 COS 访问URL |
| data.thumbUrl | String | 缩略图 URL |
| data.size | Long | 文件大小（字节） |
| data.fileName | String | 原始文件名 |

## 业务逻辑

1. **文件校验**: 非空 → 大小≤10MB → Content-Type 为 image/*。
2. **COS 上传**: 按 `bookstore/{memberId}/` 目录组织，自动生成唯一 imageId。
3. **图片关联**:
   - 传入 `borrowDetailId` + `borrowOrderNo` → 写入 `book_borrow_detail_image` 表
   - 传入 `bookId` → 写入 `book_image` 表
   - 都不传 → 仅上传不记录，由业务方后续关联
