# E06 — 微信扫码能力

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff`

员工通过微信内置扫一扫功能扫描会员的会员码（二维码），解析后获取会员ID，用于后续业务操作（借阅、积分等）的身份确认。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 解析会员码 | POST | `/api/mp/v1/staff/member-code/scan` | 是 |

---

## 扫码业务流程

```
┌──────────┐    1. 打开扫一扫    ┌──────────┐    2. 扫描二维码    ┌──────────┐
│ 员工小程序 │ ──────────────────> │ 微信扫一扫 │ ──────────────────> │  会员码   │
└──────────┘                     └──────────┘                     └──────────┘
       │                                                                │
       │ 3. 获取扫码结果字符串                                            │
       │ <───────────────────────────────────────────────────────────────┘
       │
       │ 4. POST /staff/member-code/scan {"scanResult": "..."}
       │
       ▼
  ┌──────────┐    5. 解析并验证    ┌──────────┐
  │ 后端服务  │ ──────────────────> │  数据库   │
  └──────────┘                     └──────────┘
       │
       │ 6. 返回 memberId
       │
       ▼
  ┌──────────┐    7. 跳转会员首页
  │ 员工小程序 │
  └──────────┘
```

### 步骤说明

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 员工点击"扫一扫"按钮 | 调用微信 `wx.scanCode` API 打开扫码界面 |
| 2 | 扫描会员出示的二维码 | 会员码为店面展示的 PDF417 / QR Code |
| 3 | 获取扫码结果字符串 | 微信返回原始扫码内容，格式: `MEMBER:会员编号:TIMESTAMP:时间戳` |
| 4 | 调用解析接口 | 将扫码结果POST到后端进行解析验证 |
| 5 | 后端解析验证 | 校验格式、有效期、会员是否存在、会员卡是否有效 |
| 6 | 返回会员ID | 验证通过后返回 memberId |
| 7 | 跳转会员首页 | 前端根据 memberId 调用 `/staff/members/{memberId}/overview` |

## 1. 解析会员码

```
POST /api/mp/v1/staff/member-code/scan
Content-Type: application/json
Authorization: Bearer <accessToken>
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| scanResult | String | 是 | 微信扫码获取的原始字符串内容 |

**扫码结果格式说明**:

| 格式 | 示例 |
|------|------|
| `MEMBER:{memberNo}:TIMESTAMP:{timestamp}` | `MEMBER:65000000001:TIMESTAMP:1751234567000` |

### 请求报文示例

```json
{
  "scanResult": "MEMBER:65000000001:TIMESTAMP:1751234567000"
}
```

```bash
curl -X POST "https://api.xhbookstore.com/api/mp/v1/staff/member-code/scan" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.xxxxx" \
  -H "Content-Type: application/json" \
  -d '{"scanResult": "MEMBER:65000000001:TIMESTAMP:1751234567000"}'
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 操作成功 |
| data.memberId | String | 解析得到的会员ID |

### 响应报文示例

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberId": "1"
  },
  "requestId": "req_20260630_143000_040"
}
```

## 业务逻辑

1. **扫码结果解析**: 接收微信扫码结果字符串，按约定格式 `MEMBER:{memberNo}:TIMESTAMP:{timestamp}` 解析。
2. **格式校验**:
   - 必须以 `MEMBER:` 开头
   - 必须包含 `TIMESTAMP:` 部分
   - 会员编号不能为空
3. **有效期校验**:
   - 提取时间戳，判断与当前时间的差值
   - 会员码有效期通常为 5 分钟（可配置）
   - 超时返回 `MEMBER_CODE_EXPIRED(20005)`
4. **会员存在性校验**: 根据会员编号查询会员记录，不存在返回 `MEMBER_NOT_FOUND(20001)`。
5. **会员卡状态校验**: 检查会员卡是否过期或禁用，异常状态返回对应错误码。
6. **返回会员ID**: 验证通过后返回会员的数据库ID（memberId），供后续接口使用。
