# E02 — 借阅列表页

> 对应代码: `StaffController.java` | 基础路径: `/api/mp/v1/staff`

员工查看全市（本门店）借阅订单列表，支持按手机号、订单状态筛选，点击订单可查看详情。支持办理还书操作。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 全市借阅列表 | GET | `/api/mp/v1/staff/borrows` | 是 |
| 借阅详情（员工侧） | GET | `/api/mp/v1/staff/borrows/{borrowId}` | 是 |
| 办理还书 | POST | `/api/mp/v1/staff/borrow-returns` | 是 |

---

## 1. 全市借阅列表

```
GET /api/mp/v1/staff/borrows
Authorization: Bearer <accessToken>
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | String | 否 | 会员手机号（模糊搜索） |
| status | String | 否 | 借阅状态筛选 |
| pageNo | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 20 |

**借阅状态枚举 (borrowStatus)**:

| 值 | 说明 |
|----|------|
| 0 | 借阅中 |
| 1 | 部分归还 |
| 2 | 已全部归还 |
| 5 | 已取消 |


## 3. 办理还书

```
POST /api/mp/v1/staff/borrow-returns
Content-Type: application/json
Authorization: Bearer <accessToken>
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| borrowOrderNo | String | 是 | 借阅单号 |
| returnItems | Array | 是 | 归还图书列表 |
| returnItems[].borrowDetailId | Long | 是 | 借阅明细ID |
| returnItems[].bookId | Long | 是 | 图书ID |
| returnItems[].returnQty | Integer | 是 | 归还数量 |

### 请求报文示例

```json
{
  "borrowOrderNo": "BO20260630001",
  "returnItems": [
    {
      "borrowDetailId": 1,
      "bookId": 101,
      "returnQty": 1
    },
    {
      "borrowDetailId": 2,
      "bookId": 102,
      "returnQty": 1
    }
  ]
}
```

```bash
curl -X POST "https://api.xhbookstore.com/api/mp/v1/staff/borrow-returns" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "borrowOrderNo": "BO20260630001",
    "returnItems": [
      {"borrowDetailId": 1, "bookId": 101, "returnQty": 1},
      {"borrowDetailId": 2, "bookId": 102, "returnQty": 1}
    ]
  }'
```

### 响应参数

| 字段 | 类型 | 说明 |

| code | Integer | 0=成功 |

| data.success | Boolean | 是否成功 |
| data.returnOrderNo | String | 归还单号 |

## 业务逻辑

1. **全市借阅列表**: 查询本门店（deptId）下所有会员的借阅订单；支持按手机号模糊匹配；支持按借阅状态筛选。
2. **借阅详情**: 根据借阅单号查询订单主表、明细表和归还记录表，组合返回完整订单信息。
3. **办理还书**:
   - 校验借书单号不能为空
   - 校验归还图书列表不能为空
   - 逐项校验归还数量不能超过借阅数量与已还数量的差值
   - 生成归还单号（前缀 `RT` + 日期 + 序列）
   - 更新借阅明细的已还数量
   - 若全部图书已还清，更新订单状态为"已全部归还"(2)
   - 操作人从 JWT Token 中获取
